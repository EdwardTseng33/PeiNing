# 沐寧 Munea — App 設計需求文件（Design Brief）

> 給 Claude Design 做正式視覺設計用 · 2026-06-26
> 視覺對標：Careon 官網（careon-site.vercel.app，同公司智慧健康產品線，沐寧延續其視覺 DNA）

---

## 一、產品一句話
住在手機裡、會說台語、記得你、出事會找人的「**家人替身**」。子女買給長輩（用的人 ≠ 付錢的人）。

## 二、定位
- 銀髮**個人 AI 專屬管家** App
- 核心用戶：**長輩 60+**（被服務的主人）
- 買單者：**成年子女**
- 形象：**30–35 歲專屬管家**，刻意去老人味 —— 不是「老人 App」，是「長輩的專屬管家」
- 平台：iOS 先、Android 後

## 三、App 架構（一個 App、底部兩分頁）
1. **陪伴**（長輩主畫面）— 預設
2. **親子追蹤**（子女儀表板）

**核心操作主線**：App 平常關著 → 推播去找長輩 → 點開**直接進到「管家在跟他說話」**。長輩不需要自己想到開 App、不需要學導航。畫面要繞著這條主線設計。

## 四、畫面藍圖（完整 App，不只目前原型兩頁）
1. **子女裝機引導**：選管家形象 → 填長輩基本資料 → 設用藥／事項提醒 → 填家人緊急聯絡 → 交給長輩使用
2. **陪伴主頁（長輩）**：管家形象卡 + 「跟我說說話」大鈕 + 今天要記得 + 找家人
3. **對話進行中**：全屏管家臉 + 即時字幕 + 結束鈕（長輩版超大字）
4. **推播樣式（鎖屏／通知）**：管家臉縮圖 + 一句話 + 快速鈕（「吃了」「等等再說」）+ 緊急可破靜音
5. **親子追蹤（子女）**：今日狀況四格（對話次數／用藥回應／情緒／最後互動）+ 今天聊了什麼（管家彙整）+ 提醒管理 + 選管家
6. **找家人／緊急**：一鍵通知家人

## 六、管家形象（關鍵 · 上一版最該升級處）
- **精緻真人質感**（30–35 歲管家氣質），**不是表情符號／廉價卡通**
- 提供 **4 個形象：2 男 2 女**，長輩二選一
- 起手兩位：**Aria**（女·陪聊／情緒）、**Aiden**（男·用藥／緊急）
- 對標「會說話的臉」競品水準：**Duix / LiveAvatar / Anam / Tavus**
- 兩軌並行：① 擬真真人（達競品水準，待硬體測試）② 2D 精緻角色（暖但不幼稚）
- 說話時臉／嘴會動；只在 App 開啟時呈現（推播只用靜態臉縮圖）

## 七、無障礙（長輩優先）
- 大字（主畫面內文 ≥ 19px、主要按鈕 ≥ 20px）
- 高對比、暖色不刺眼
- 少元素：一個畫面一件主要的事
- 大圓鈕、好點、容錯

## 八、法規紅線（設計文案必守）
- 用藥提醒 = **鬧鐘**，不提供個別用藥建議（碰了 = 變醫療器材要查驗登記）
- 情緒陪伴**不能稱「諮商／治療」**（心理師法）
- 危機 → AI 是**轉介者**（1925／119／通知家人），不自己扛

## 九、現況
原型已用網頁做出陪伴頁 + 親子追蹤頁（療癒綠 + 卡片 + 底部分頁 + 會動的管家臉），程式在 `web/`。本文件供 Claude Design 接手做正式視覺設計與完整畫面。
