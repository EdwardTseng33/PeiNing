## 0. 一句話目標
**聊聊 = 住在手機裡、像視訊通話一樣的即時語音陪伴：會記得你、懂你此刻的處境、也在乎你在乎的家人。非醫療；出事會找人。子女買、長輩用。**

## 1. 成功長什麼樣（產品目標）
1. **開口就聊**：像打電話，低延遲、可自然打斷（不是按鈕一問一答）
2. **記得你**：跨對話的長期記憶（你說過的人事物、習慣、心情）
3. **懂此刻**：時間、天氣、在地、你的近況都納入回應
4. **有個性**：6 角色，各自「一張臉＋一個聲音＋一種個性」
5. **安全**：危機（自傷／急症）偵測 → 溫柔轉介家人／1925／119
6. **一張會動的臉**：先 2D 保底、後擬真（Ditto／LiveAvatar）

## 2. 整體結構（分層）
```
┌── 前端：全屏管家臉、視訊通話感、超大結束鈕 ──┐
│            ↕  即時語音（WebSocket 串流）        │
│  反射腦 = 即時聽說（Gemini 3.1 Flash Live）    │
│                                                 │
│  背景腦：管家腦（整理記憶／主動關心）           │
│          守護腦（危機安全網）                    │
│                                                 │
│  橫向三支柱：角色人格層 · 記憶層 · 感知層       │
│                                                 │
│  服務組裝層：回應 = 人格＋記憶＋感知＋當下對話  │
│              ＋安全規則＋語音表達限制           │
│                                                 │
│  資料層：Supabase（正式）／本機 JSON（過渡）    │
└─────────────────────────────────────────────┘
```

## 4. 回話怎麼組成（資料流）
`使用者說話` → 服務組裝層叫出〔角色人格＋相關記憶＋當下感知需求＋守護腦安全判斷〕→ 拼成一份指令 → 語音大腦生成 → 即時語音回覆。
公式：**回應 = 角色人格 ＋ 使用者記憶 ＋ 即時感知 ＋ 當下對話 ＋ 安全規則 ＋ 語音表達限制。**

## 5. 技術棧（詳見 `技術棧與工具總覽.md`）
- **大腦**：Gemini 2.5 Flash（文字）｜Gemini 3.1 Flash Live（即時語音）｜Claude Sonnet 5（管家／守護）
- **語音**：Gemini Live ＋ Voice Provider Adapter（可替換層）
- **臉**：2D viseme 保底 ／ Ditto ／ LiveAvatar（RunPod，已付費測試）
- **前端／殼**：Capacitor + Web Core
- **資料**：Supabase（Postgres + RLS）／ JSON 過渡
- **部署**：Vercel（網頁）＋ 小型雲端主機（即時語音長連線）
