  - ⚠️ AI 生成臉瑕疵：**五官歪/不對稱** → landmark 抓錯 → 動起來抽搐（要丟）；**塑膠美顏** → 踩恐怖谷（prompt 已寫 NOT over-retouched，別讓工具自動美顏）。手指/耳朵畸形不影響（被裁掉）。

## 軌道二/三 · 2D 卡通 + 動物 ⚠️ 先別狂生、先 PoC

**核心矛盾**：圖片生成出「一張平的圖」；Live2D 要「分層 PSD（眼/嘴/髮分開+補畫遮擋）」才能 rig。**平面圖不能直接動。** Live2D 官方 Cubism 5「Material Separation Plugin」只是**半自動加速、仍需插畫師/rigger 收尾**——拆層補畫是真實人力成本，6 角色是 v1 最大黑洞。

**三條路（按務實度）**：
- **路 A（CTO 最推 v1）**：2D/卡通也走「**單圖 talking-head**」（Ditto 本身或 Talking Head Anime 4）→ **跳過 Live2D 拆層**、6 角色一條管線。變數＝insightface 對「大眼扁平卡通臉」landmark 偵測準不準（未知）→ **要 PoC**。
- **路 B（女巫原案）**：full Live2D rig——最自然、最重最貴，畫風定稿後確定長期投資再做（列 v2）。
- **路 C（v1 最省）**：靜態表情圖切換（中性/微笑/關心/說話）+ 簡單嘴開合，**複用引擎現成 AvatarStage+lipsync 架構**（deadends 標的已驗證路線）。非即時逐句對嘴，但「會動+嘴開合」長輩 v1 夠用、白做風險最低。

**動物**：人臉偵測器大概認不出貓狗臉（路 A 大概率不通）→ v1 走路 C（靜態切換）或路 B（Live2D，動物適合 Q 版擺動）；動物的「動」＝耳/尾/頭歪，不靠對嘴。

**🔑 防白做閘門**：**先花 ~30 分鐘、<NT$30 跑「卡通臉/動物臉餵 Ditto」PoC**（RunPod 開回暫停的 pod、卡西法導）：
- 通 → 2D（也許動物）走 Ditto 同管線、Live2D 重工全省。
- 不通 → 2D/動物走路 C（引擎現成）撐 v1、Live2D 列 v2。
- **PoC 出結果前，不照「Live2D 平面圖」思路砸錢生一堆。**
