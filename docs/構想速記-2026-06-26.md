## 1. 構想一句話

用 BeyondVoice 的「語音 + 一張會說話的臉 + 會記事的腦」，做一個專給長輩的 App：**陪聊、顧情緒、提醒吃藥**。

## 2. 為什麼契合 BeyondVoice 底層（比創作者分身更天然）

| 銀髮需求 | BeyondVoice 現成的底層 |
|---|---|
| 長輩不打字 → 要用講的 | OpenAI Realtime 即時語音（耳朵+嘴+閒聊腦） |
| 孤獨 → 要一張有溫度、會回應的臉 | 2D 數位真人 avatar |
| 要記得長輩說過的話、察覺情緒變化 | Claude Sonnet 4.6 深想腦 |
| 到點主動開口提醒吃藥 | 城堡派工 / 主動觸發底層 |

→ 策略洞察：**BeyondVoice 從「一個產品」升級成「能長多個垂直產品的引擎」**，銀髮陪伴是第一個垂直。引擎共用、但這個垂直需要自己的合規骨架。

## 3. 最大的坎（誠實 · 這是 go/no-go 關鍵）

銀髮醫療的監管 + 責任，比 BeyondVoice 主線（創作者）重一整個檔次：

- **用藥**：「提醒吃藥」（鬧鐘 + 陪伴語氣、相對安全）vs「建議用藥 / 劑量 / 衛教」（醫療行為、踩法規紅線）——這條線怎麼劃，決定產品能做多深。
- **情緒照護**：對象可能憂鬱 / 孤獨 / 失智 / 有自傷風險，AI 講錯一句話的責任很重。要有安全網（偵測危機 → 通報家人 / 專線，不是 AI 自己扛）。
- **健康個資**：最高敏感等級。長輩本人同意 vs 子女代理怎麼處理。

→ 沙利曼（信任 / 合規）從第一天就要在。

Careon = 一套「軟硬體智慧照護員」大方案（Home Pad 硬體 + 端側運算 + 衛星感測器 + Family App + 穿戴整合）。

為什麼這打法對：硬體要供應鏈 + 認證 + 生產，重又慢；純 App 上架快、成本低、先驗證市場，未來再長硬體。

### Careon 已鋪好、直接站肩膀上的

| 資產 | 內容 |
|---|---|
| 2 個 AI persona | Aiden（男·用藥/健康/緊急）+ Aria（女·陪聊/情緒/衛教） |
| 5 個用戶 Tier | 重度長者 / 預防型 / 子女買單者⭐ / 高薪 / 機構 + persona + 痛點 + 定價（NT$499-1299/月） |
| 核心洞察 | 用的人 ≠ 付錢的人（長者用、子女買） |
| 心理照護 | 三層感測（語音/視訊/行為）+ 主動陪聊 + 危機導入 1925 + 隱私守則 |
| 競品 | Ato / ElliQ / Hyodol（都沒做主動心理感測） |

### App-only 版 vs Careon 軟硬體版的取捨

- 沒 Home Pad = 沒硬體感測器（跌倒 / 環境 / PIR）。App 版「感測」靠手機 + Apple Watch / HealthKit。
- App 做得到：avatar + 語音陪伴 + 用藥提醒 + 情緒陪伴 + 家人儀表板。
- App 版先放掉：跌倒偵測、衛星感測器、CGM 整合（要硬體 / 未來）。

## 7. 沙利曼法規紅線（已盤完 · 🟠 GO-with-紅線）

- **用藥**：只當「鬧鐘 + 陪伴」，不當「藥師」（碰個別用藥建議 / 劑量 = 變醫療器材 SaMD 要查驗登記）
- **情緒**：陪伴可以，不能稱「諮商 / 治療」（心理師法）；危機 → AI 是轉介者（1925 / 119 / 通知家人）不是治療者
- **健康個資**：特種個資、失智長輩同意能力是最硬的坑
- **第一版安全範圍**：陪聊 + 純鬧鐘用藥提醒 + 情緒陪伴 + 安全網 + 家人儀表板
- Careon 原規劃本就有合規意識（非醫療診斷 / 視訊預設關 / 語音不上雲）= 跟沙利曼盤的一致

## 11. 卡西法技術盤點（已完成 · 結論：可行、比想像快）

- **技術路線**：Capacitor 把現有 BeyondVoice 網頁「裝進 iPhone 殼」——前端已是 PWA-ready 網頁版 App（WebRTC 即時語音 + service worker + manifest 都在），最難的語音引擎 + 後端整套搬過去不重做。
- **第一版 ~70% 複用**：語音陪聊 / 會說話的臉 / 情緒陪伴 / 記憶 / 登入都現成。要新做：銀髮大字 UI 殼 + 用藥提醒（純鬧鐘）+ 家人儀表板 + 危機安全網 + Aiden/Aria 角色。
- **Apple 上架最大雷 = 用藥那條線**：守住「鬧鐘 ≠ 藥師」（沙利曼已劃）+ 文案不出現任何醫療宣稱（治療 / 諮商 / 改善憂鬱都不行）→ 過審機率高。隱私架構（鏡頭預設關 + kill switch）是審核加分。
- **工時**：active 64-108 hr / 曆時 2-3.5 週（含 Apple 審核來回 + avatar/配音）。
- **第一步建議**：先做「能講話 + 能設提醒」最小可跑版上真機，釘死「語音延遲可不可接受」這唯一技術未知數。
- 完整：卡西法盤點（對話紀錄）
