## 0. 一句話定案
感知層照 **「清晨背景先備好 → 開場就講出來 → 通話中只讀已備好的、絕不臨時對外查」** 這條路做（避開即時語音卡頓＋幻覺）；地基＝**中央氣象署天氣＋環境部空氣品質＋當地時間**（都免費／官方／即時算）；靈魂＝**會挑對時機的主動開口**（借 ElliQ「先算了才開口」）；**語氣與情緒感知＝基礎感知的一部分**（Edward 2026-07-02 拍板：智慧照護 App 的感知必含此能力）——聽出心情調整寧寧的溫度、餵主動開口、給家人溫和觀察，**但絕不診斷**；**V2 視訊／表情**進 Backlog 預備、目前不開發，以「同一種訊號格式」預留、之後插件式接上、不重寫。

## 1. 架構定案（為什麼這樣切）

### 1.1 鐵律：背景預抓、開場注入、通話只讀本地
2026-07 技術調研的關鍵發現：**Gemini 3.1 Flash Live 的工具呼叫是「同步阻塞」**——模型呼叫外部工具會停下來等回傳才續講；且**不支援 Maps grounding**。對長輩＝通話中臨時查天氣會讓寧寧「講到一半卡住」（體驗災難）。

因此感知層四步（固定流程）：
```
[清晨排程 · 長輩起床前 e.g. 06:30]
  抓 CWA 天氣(該長輩縣市) + 環境部 AQI → 便宜文字腦(Gemini 2.5 Flash)生一句「今日簡報」
  → 存 perception_snapshot{personId, date, weather, aqi, briefingLine, expiresAt}
[長輩按「聊聊」]
  開場脈絡注入 briefingLine（1-2 句人話、非整包資料）→ 寧寧開場即「阿嬤早，今天冷、加件外套」
[通話中若問天氣/空氣]
  同步工具只讀 Supabase snapshot（毫秒級）→ 不打外部 API
  snapshot 過期(TTL)才由背景刷新
```

### 1.2 雙模型逃生門
主力 **Gemini 3.1 Flash Live**（最便宜、延遲夠、繁中可用）續用；但把 S2S 呼叫封成抽象層，保留 **Gemini 2.5 Flash Native Audio**（唯一支援「背景非阻塞工具」）當將來需要「工具在背景跑、寧寧同時繼續聊」時的切換選項；OpenAI GPT-Realtime 當品質備援（貴約 10×，非台灣腔強項）。

### 1.3 修掉現有兩個斷點（P0 前置）
- **孤兒抽屜**：`perception_snapshots` 存取都在、但 `build_reply_context` 從不讀回 → 先接回（1 行架構級斷點，不補則後面接再多來源都白接）。
- **假天氣**：拿掉 `chat_engine` 寫死的「今天轉涼有寒流」。

## 4. 主動開口＝感知層的靈魂（借 ElliQ「先算了才開口」）

競品調研結論：全球做「主動關心」最好的是 ElliQ——**不是死板定時，而是先算一個「該不該現在出聲」的分數**，夠高才開口。這是讓寧寧顯得體貼而非煩人的關鍵，華語市場沒人做＝我們的護城河。

**寧寧的主動開口引擎（定案）**：
- **開口分數 = 估計心情 × 有沒有空/作息 × 最近被歡迎程度**，分數夠高才主動。
- **多元觸發、不只看鐘**：時間/作息（學到個人化，如「他晚上 7:30 比較願意聊」）＋ 世界變化（天氣轉冷、節氣、在地新聞）＋ 記憶回呼（上週提的孫子考試 → 追問）＋ 心情訊號（昨天聲音低 → 今天更溫柔開場）＋ 關懷目標（喝水/用藥，但包成關心不是嘮叨）。
- **防煩四條**：相關性＋時機＋好退出（「少一點／換時間」）＋透明（說明為何找你）；有頻率上限＋回饋迴路（被忽略/縮短 → 自動退頻）。
- **節奏甜蜜點**（沿用已上線語音陪聊服務）：每天、每次 10-15 分、給家人一則事後摘要。**每天但「有分寸且短」勝過「頻繁又籠統」。**

## 5. 語氣與情緒感知（基礎感知能力 · Edward 2026-07-02 拍板）＋ 非醫療紅線（憲法級）

我們是語音優先，**從聲音聽情緒**是天生強項；作為 AI 智慧照護 App，**這是基礎感知的一部分、不是選配**。分兩步收，兩步都在 V1 範圍：

- **第一步（隨 P0/P1 就有）**：即時語音模型本身就聽得懂語氣——在寧寧的人設脈絡明確要求「**聽出對方的語氣與心情、跟著調整**」（聽起來累→放柔放慢；開心→跟著亮起來；低落→不催、多陪）。零延遲、零新件。
- **第二步（P1 · 正式情緒感知模組）**：聊完後由背景管家腦對**這輪對話**做情緒判讀——**三個角度：語氣（怎麼說）、用語（慣用的說法/句型）、用詞（選了什麼字眼）**——粗略分級＋信心度，存成 `WellbeingSignal`（見 §7 統一格式）進**時序趨勢庫**（每位長輩有自己的基準線）。用途四個：① 餵**主動開口分數**（昨天聲音低→今天更溫柔、更早關心）② 餵**活的側寫**（心情走向）③ 給家人的**溫和觀察**（「這週聊得比較少、聲音比較低」——是觀察、不是判定）④ **App 內「心情數據版」**（Edward 2026-07-02 拍板：每次對話的情緒分析要記錄下來、在 App 裡以看得懂的指標呈現——顯示方式經專項調研後定案，見 `情緒數據版-顯示調研`；鐵則＝溫暖觀察、不是醫療報告、不像監控）。
- **技術選擇**：優先用現有大廠語音/文字模型自帶能力（Gemini 對語氣的理解）判讀；若將來要離線/更準，備選 SenseVoice-Small（中文強、可在裝置上跑）——**不綁脆弱第三方**（市場警訊：純語音生物標記公司 2026 在收——Kintsugi 倒、Hume 表情 API 六月退場）。
- **可做（companionship）**：用語氣調整寧寧溫度；心情趨勢餵主動關心；把「這週聊得少、聲音低」當**觀察**溫和告訴家人；追蹤**自述**心情趨勢。
- **❌ 絕不做（=醫療器材 SaMD，觸法）**：說/暗示「你的語音顯示憂鬱/認知衰退徵兆」、任何當成篩檢的風險分數、以疾病為由叫人看醫生。**連內部標記「疑似認知衰退」給家人都算診斷責任**——一律禁止。`is_medical_inference` 硬閘（§7.3）從第一天就上。

## 6. 信任與隱私定案（子女買、長輩用）

這是我們的商業模式成敗與最尖銳的張力。

- **同意人是長輩本人、不是付錢的子女**：定案「**家長主導、子女協助**」——子女可送裝置，但**啟用要長輩自己在裝置上點頭**；子女只是「有範圍的提醒接收人」、不是原始資料觀看者。**只有子女點頭＝無效同意＝我們最大法律曝險。**
- **給觀察、不給原始資料**：家人看到的是「聊得開心／這週比較安靜」摘要與真關心警示，**不是逐字稿、不是錄音**。
- **對長輩透明**：長輩知道寧寧會給家人一則溫和週報、且可看可限。
- **V1 不用鏡頭、不常開麥克風錄環境**：只在「對話中」用語音。台灣/華語市場正走向「無鏡頭、無環境錄音」隱私優先設計。
- **台灣法規（2026 已變嚴）**：個資法 2025 修法設個資會、72 小時通報、重罰至 NT$1500 萬；健康＝特種個資（第 6 條）需書面目的同意。**健康/情緒訊號一律當第 6 條特種個資對待。**
- **文化定案點**：台灣長輩壓倒性偏好**卡通/角色化形象（82.9%）**勝過打馬賽克真人影像；隱私是 100% 未採用者的門檻 → **角色化 avatar 的產品在台灣勝過真影像產品**（正好呼應我們寧寧角色化路線）。

## 7. V2 預留（視訊動作＋表情監測）——現在設計、之後插件

### 7.1 誠實的成熟度判定（別踩雷）
- ✅ **成熟可上、但誤報多**：跌倒/活動/在場偵測（SafelyYou 自己數據：**62% 被判「跌倒」其實不是** → 誤報是信任殺手第一名 → 要人工複核或多感測融合）。
- ✅ **隱私逃生門**：**毫米波雷達（Vayyar/Soli 類）**做跌倒/睡眠/呼吸——**不產生影像**、暗處可用，是「子女看父母」最強信任故事，避開生物特徵法規。
- 🔴 **不成熟＋法律地雷**：「從臉/攝影機判情緒/憂鬱」對個人＝科學有爭議（Barrett 2019：臉部表情≠情緒指紋）、微軟已下架情緒 API、Hume 表情 API 退場、EU AI Act 列高風險。**表情情緒只能當「離線、粗略、聚合」的悄悄訊號，絕不當對家人的情緒標籤或賣點。**

### 7.2 V2 順序定案
**先做安全（跌倒/活動，用雷達或離線姿態、框成「安心」）→ 表情情緒最後、且各自獨立同意。** 這樣法律與市場雙重降險。avatar 會動的臉可搭 Ditto（開源自架 RunPod）或 HeyGen LiveAvatar（最快商用），**角色化＞擬真**（避恐怖谷＋台灣偏好）。

### 7.3 現在 V1 就要預留的五個抽象（讓 V2 是插件、不是重寫）
1. **統一「健康訊號」事件格式 `WellbeingSignal`**：不管來源是語音(V1)、語氣(V1)、表情(V2)、動作/跌倒(V2)，都吐同一種形狀：`{時間, 來源modality, 訊號type, 值, 信心度, 來源模型, 在裝置或雲端, is_medical_inference=false}`。上層（寧寧腦、家人看板）只吃這個格式、不碰原始音訊/影像。
2. **同意能力清冊 Capability Registry**：每個感知能力（voice_emotion / face_expression / fall_detection…）**可獨立同意、獨立開關**；V1 先掛語音類，V2 新增只是註冊新能力、各自要長輩點頭。
3. **裝置邊界線**：原始訊號（現在音訊、將來影像）**不出裝置**，只有抽象化的 `WellbeingSignal` 可離開；用雲端語音模型＝明確記錄的例外。V2 鏡頭沿用同邊界（影像在裝置處理、只吐抽象事件）。
4. **時序趨勢庫**：訊號存時間軸、每位長輩有**個人基準線**（「這週比平常安靜」是相對、不是絕對）；只有趨勢（不是單次）驅動關心與家人摘要。
5. **`is_medical_inference` 硬閘＋翻譯層**：任何構成診斷/篩檢的訊號一律擋下（V2 誘惑往richer inference 時守住非醫療）；翻譯層把訊號轉成溫暖非診斷語言給長輩、轉成有範圍摘要給家人、**絕不吐「憂鬱」這種標籤**。

## 8. 健康資料放哪層（沿用 2026-07-02 已定邊界）
- 🧠 進**記憶**（長期、記趨勢）：慢性病、運動/飲食習慣、回診遵從、心理狀態趨勢。
- 🌤️ 進**感知**（即時、會過期）：今天步數/血壓/睡眠、今天有沒有回診、當下心情。
- 🔗 橋接：感知的今日數據若重複/重要 → 整理員沉澱成記憶的長期趨勢（連日血壓偏高 → 記成長期狀況 → 主動關心）。

## 11. 我們 vs 市場（定案結論）
- **設計贏在貼合**：voice-first ＋ 真懂長輩世界（天氣/在地/記憶）＋ 會挑時機主動 ＋ 家人信任層 ＋ 繁中在地——四件全包，華語市場沒人做。
