# 第 1 章 · 願景與定位（北極星 · 不漂）

## 1.1 一句話遠景
讓一個人覺得：螢幕裡這個，**像一個真的住在身邊、會說話、記得我、懂我、出事會找人的存在**——不是 App、不是玩具。

## 1.2 官方一句話定位〔已定 · Edward 2026-06-28 拍板鎖定〕
> # 「會記得你，也在乎你在乎的人。」
> 所有設計／行銷／文案對齊這一句，不再各說各話。內部敘事三種講法同向：「家人替身」/「專屬管家」/「在乎你的人」——對外只鎖這一句。

### 1.2.1 這句話要撐起的「靈魂」（Edward 親述 · §7 對話智能 / §8 行為層 必須 honor）
記憶＋關係、溫暖關愛、**有靈魂、像個真正的朋友／真正的家管／貼心的照護**。
**關鍵差異化（跟市面聒噪 AI 的根本不同，這是護城河）：**
- **懂得什麼時候該講、什麼時候該靜**——有社交分寸，不嫌煩、也不聒噪。
- **能懂你**——讓人驚豔的「感知 ＋ 聊天互動」能力：接得住情緒、記得脈絡、主動但不打擾。
> 設計鐵則：寧寧的價值不在「話多」，在「**懂得拿捏**」。靜得恰到好處，比一直講更像真人。

## 1.3 三客群 × 同一個產品（不貼身份標籤）
不按身份分類，按「關係」講話——三種人渴望相同：有一個真的在乎的存在。
- **A 需要陪伴與日常照護的人（第一情境含長輩）**：痛＝沒有人好好接住日常、情緒與健康節奏。
- **B 慢性病日常者（隱性、不貼病人標籤）**：痛＝一輩子的功課、不想被當病人盯。
- **C 需慰藉者（照護者&被照護者雙向）**：痛＝失掌控感/照顧到掏空。
- **適用對象＝8 歲到 80 歲都成立**（角色 v2 放寬：真人角色不鎖輩分、是「最在乎你的那個人」）。

## 1.4 達標標準（怎麼算做到了）
- **臉**：擬真軌盲測對標 Tavus/Anam 不露餡；2D 軌可愛、嘴型順、不恐怖谷。
- **對話**：被打斷反應像真人（不硬停）、接得上上文、聊完覺得「像跟一個人說話」。
- **感知**：早晨第一句自然帶出「今天」（天氣/重要日子/家人），非罐頭問候。
- **記憶**：跨天記得上次的事（「上次你說膝蓋痛，好點沒」），Day 30 還黏著。
- **延遲**：一來一往順、不卡。

# 第 2 章 · 目標用戶與場景

## 2.1 三個使用視角（缺一不可）
- **本人使用者**：每天打開首頁與 `聊聊`，獲得陪伴、健康日常提醒、記憶延續與情緒支持。可以是長輩，也可以是任何需要被陪伴與健康照護的人。
- **家人/照護者**：關心、互動、接收重要提醒、查看被授權的健康與日常狀態。
- **高價值第一場景：長輩照護**：子女代裝、家人付費、本人簡單使用仍是 v1 的重要 wedge，但不能把產品定位寫死成單一年齡族群。

## 2.2 產品死穴（首頁、聊聊、家人互動必須一起設計）
1. 首頁如果像健康報表，`聊聊` 會被降級成附屬功能。
2. 依賴使用者記得開 App＝日活爛掉（**主動關懷與推播是命脈**）。
3. 家人互動若只是監看，會變成壓力；必須是支持、鼓勵、傳話與一起完成。
4. 健康照護若太醫療化，會踩法規與心理壓力；必須守住「陪伴與生活提醒」。

## 2.3 用戶旅程關鍵
- **廣告/對外敘事**：主軸是「AI 健康照護、有人記得你、也在乎你在乎的人」。家人照護可作第一支情境，不等於全部產品。
- **Day 1**：主動出聲 + 第一句話＝留存決定性。
- **Day 8-30**：記憶連續性跨過 Day 30 大坎。
- **持續**：子女儀表板有內容＝續訂；空白＝退訂。

# 第 4 章 · 系統架構（三顆腦 + 一張臉）

## 4.1 三顆腦
- 🧠 **反射腦**（對話當下·要快）：自然聽講、被打斷不硬停（聽到→判斷→可能補半句→接話記上文）。低延遲是命脈。
- 🧠 **管家腦**（背景·深想·不擋路）：感知（清晨備今天）+ 記憶（深度認識這個人）。**深功夫背景做 = 深而不慢。**
- 🧠 **守護腦**（安全網·P0）：偵測異常/危機 → 通知家人/轉介。

## 4.2 ⭐ 反射腦引擎 = Gemini 3.1 Flash Live〔已定 · 推翻 gpt-realtime-2〕
**語音大腦一律 Gemini 3.1**（產品對話 `gemini-3.1-flash-live-preview`）。
- **背景**：6/27 上午曾拍 gpt-realtime-2、**6/27 深夜 Edward 親耳改 Gemini 3.1**（30/30/30/10 秤、比 GPT 便宜 ~3 倍、台灣中文更自然）→ 推翻舊成本表假設。
- **⚠️ 全域改正**：規格書/avatar藍圖/商業v3/定案/Sprint1 施工表凡寫「gpt-realtime-2」者，一律以本章 Gemini 3.1 為準（舊文件待回寫）。
- 〔待 Codex 校〕Gemini 3.1 Flash Live 的 audio 串流接口 vs 原 gpt-realtime 接法差異、AOL 餵 Ditto 那段是否需改（盤點缺口 G-3）。

## 4.3 背景管家腦選型〔建議·待卡西法成本細算〕
不是一顆大模型——是「排程代碼 + 規則 + 需判斷時才叫便宜 AI（**Gemini Flash-lite 級**）」的組合：
- 純代碼（不花 AI 錢）：到點提醒、健康同步、門檻告警。
- 叫便宜 AI（少量）：晨間 brief 措辭、情緒判讀、記憶整理萃取。

## 4.4 兩交互層〔已定〕
- **App 沒開 → 推播層**：標準 iOS 推播（一定到、零 iOS 限制）。帶臉縮圖、快速鈕、緊急可破靜音。
- **App 開 → 對話層**：完整即時語音 + 會說話的臉。臉只在 App 開著時跑（省成本）。
- **死穴位移**：從「技術（能不能出聲）」→「行為（使用者會不會因為關心而點推播開 App）」——這是第一要驗的行為指標。

## 4.5 自架＝商業模式的根
整條鏈自架、零第三方每分鐘費（租第三方臉每人每月約 NT$900 一張臉 = 毛利必死）。Gemini 3.1 語音走用量計費（上線前過 license/zero-retention 帳）。

## 4.6 端到端服務流
```
              ┌─ 即時語音 Gemini 3.1 Flash Live（聊天才開·WebRTC 音雙向）
   App ───────┼─ 生臉 Ditto GPU（Modal scale-to-zero·idle 不燒）
 (iPhone)     └─ 背景管家腦（排程 24h）
                      │
        ┌─────────────┼──────────────┐
     記憶庫        健康數據          守護告警
        └──────── 子女 App（家人儀表板）┘
```
- 平台：混合原生 Capacitor（包 castle-voice-engine 網頁核心 ~70% 複用）；iOS 先、Android 後。
- 延遲預算：開口目標 <1.5 秒〔換 Gemini 後重估·待實測〕。
- 斷網降級：連不上時老實說「現在連不上、等等再陪你」，不假裝不卡死。

- 一開動態會說話的臉（2D 或擬真都一樣走 Ditto）＝扣點。

- ⚠️ 三未驗 keystone（見 §12）：Ditto TRT 25fps / LiveAvatar 45fps / 冷啟動秒數。

# 第 6 章 · 核心畫面互動規格 ⭐⭐⭐（最重要 · 補做歪根因）

> **盤點發現**：對話畫面做歪不是執行者粗心，是**根本沒有畫面級 spec**——簡報只一行藍圖。本章把它釘死到「照著做不會再歪」。
> **Edward 2026-06-28 拍板**：對話＝**一個全屏臉，就一個**。把現有「聊聊小圓臉頁 + 視訊大臉頁」收斂成這一個。

## 6.1 對話畫面 = 單一全屏管家臉〔Edward 親口拍板 · 鐵律〕

**唯一正確版面（取代現有聊聊頁 + 視訊頁）：**

```
┌─────────────────────────┐
│                         │
│                         │
│     【管家的臉】          │  ← 全屏 · 臉佔畫面主體（≥ 50% 縱向）
│    （會動時對嘴/呼吸）     │     置中偏上、像跟家人視訊
│                         │
│                         │
│                         │
│  ┌───────────────────┐  │
│  │  即時字幕（大字）    │  │  ← 臉下方、半透明底襯、字 ≥ 19px
│  └───────────────────┘  │
│                         │
│      ●  結束             │  ← 超大結束鈕、≥ 20px、好點、容錯
└─────────────────────────┘
```

**鐵律（違反＝又做歪）：**
1. **臉是全屏主體**，不是小圓頭像、不是大留白。臉佔縱向 ≥ 50%（對齊美術規格「臉佔 ~50% 畫框」）。
2. **就一個對話畫面**——不分「聊聊（小臉）」和「視訊（大臉）」。日常陪聊和「見面」是同一個全屏臉畫面。
3. **臉會不會動是「加成」不是「版面」**：有算力（Ditto/2D 引擎接好）→ 臉對嘴會動；沒接好/降級 → 臉靜態或只呼吸。**版面一律全屏臉、跟臉動不動無關。**

## 6.2 四種對話狀態（各長怎樣 · 執行者照做）

| 狀態 | 臉 | 字幕 | 其他視覺提示 |
|---|---|---|---|
| **待命**（剛進畫面/沒人說話）| 全屏臉 · idle（眨眼、呼吸、偶爾微笑）| 隱藏 或 顯示「我在聽，你說～」提示 | 結束鈕常駐 |
| **聆聽**（使用者在說話）| 全屏臉 · 專注神情（微前傾、看著你）| 顯示使用者語音轉文字（可選）| 麥克風收音指示（柔和波形/光點）|
| **思考**（送出後等回應）| 全屏臉 · 思考感（眼神微動、不空白凍住）| 「嗯…」或柔和等待提示 | 不可轉圈空等；< 1.5s |
| **說話**（管家回應）| 全屏臉 · **嘴型對嘴開合**（會動時）| **逐句顯示管家說的話**（大字、跟著語音走）| 結束鈕常駐 |

- **狀態轉換要平滑**：idle ↔ 說話接縫不可「臉跳一下」（姿態錨定，技術藍圖 §A.1）。
- **思考態最關鍵**：北極星要求「不卡」——**永不准使用者盯轉圈等 GPU**，臉要有「她在想」的活著感，不是凍結。

## 6.3 即時字幕呈現
- **位置**：臉下方、不遮臉。
- **字級**：≥ 19px（健康照護場景的大字底線）；管家說話字幕可更大。
- **呈現**：管家說話**逐句/逐字跟著語音浮現**（不是一次跳一整段），使用者跟得上。
- **底襯**：半透明深色襯底確保任何臉色背景下都讀得清（高對比）。
- 使用者語音轉文字（自己說的話）＝可選顯示、預設可關（避免畫面太滿）。

## 6.4 結束鈕
- **超大**（≥ 20px 字 + 大可點區）、常駐、永遠在固定位置（降低操作負擔）。
- 文字明確「結束」/「掰掰」，配清楚 icon。
- 容錯：誤觸有確認 或 可快速重開。

## 6.5 「會動的臉」佔位與降級（三層 · 對齊算力現實）
臉的動態程度依引擎就緒度降級，但**版面永遠全屏臉**：
1. **靜態臉**（引擎沒接好/PoC 未過）：全屏靜態臉 + 字幕（先能上線）。
2. **呼吸/眨眼**（idle-loop 預生影片）：臉會呼吸眨眼、不對嘴（成本近零）。
3. **真對嘴**（Ditto/LiveAvatar 接好）：臉跟著語音對嘴開合（目標態）。
- **降級鐵律**：算力不足/冷啟動慢時，**寧可臉不動、不可對話中斷**（「對話不斷」優先於「一定會動」）。

## 6.6 5 分頁架構〔已定 · 取代簡報「兩分頁」〕
App 底部 5 分頁 + 家人端另一視角：
- **寧寧**（首頁）/ **狀態** / **聊聊**（中央凸起=命脈「主動開口」）/ **家人** / **設定**
- 「聊聊」點進去＝§6.1 的單一全屏臉對話畫面。
- 家人端＝`family.html` 另一視角（看被授權的狀態/儀表板/傳話），不在這 5 分頁內。
- ⚠️ 簡報 §三「兩分頁」過時、以本節 5 分頁為準。

## 6.7 健康照護可用性（全 App 鐵律）
- 大字：內文 ≥ 19px、主要按鈕 ≥ 20px。
- 高對比、暖色不刺眼。
- **少元素：一個畫面一件主要的事**。
- 大圓鈕、好點、容錯。
- 推播樣式：臉縮圖 + 一句話 + 快速鈕（「吃了」「等等再說」）+ 緊急可破靜音。

# 第 7 章 · 對話智能與記憶

## 7.1 對話節奏（北極星 B · 像兩個真人）〔願景達標標準〕
- **自然被打斷**：聽到 → 判斷要不要停 → 停了也可能補完半句 → 接話記上文（不一刀切）。
- **停下後連貫**：理解上下文接話、不重新開始。
- **聽講像兩個真人**：不是一問一答的機器。
- 〔待技術落實〕這層「對話節奏設計」目前無技術文件落實怎麼做（盤點缺口 G-2）。

## 7.2 四層記憶 + 重要分數
| 層 | 記什麼 | 多長 |
|---|---|---|
| 🪨 核心檔案 | 你是誰（名字/家人/慢性病/老伴走了）| 永久 |
| 📖 長期回憶 | 重要事件/喜好 | 長期+會合併 |
| 🗓️ 近況 | 最近在幹嘛/心情 | ~1-2 週滾動〔轉鈕待 Edward 定窗口〕|
| ☀️ 當天狀態 | 今天的世界（寒流/回診/新聞）| 每天刷新 |
- 重要分數＝情感重量 + 健康相關（一律記）+ 重複出現（升級）+ 一次性閒聊（淡掉）。
- **整理員**：定期合併重複、濃縮舊的、丟低分的（避免自打嘴巴）。

## 7.3 活的側寫 + 主動開口
- **不讓用戶填表**，從聊天自動認識你、每週更新側寫〔差異化賣點〕。
- 興趣權重（常提+分、沒提退分）；改正/反向機制（說不喜歡降權、矛盾用新蓋舊、糾正改檔）。
- **主動開口**（命脈）：先開口關心 + 帶自己的話題 + 雙向討論。料來自記憶+今日感知+配興趣新發現。有分寸不煩。

## 7.4 每日感知排程
每天清晨背景跑一次：天氣（寒流→添衣量血壓）、今天回診/用藥/重要日子、挑過在地新聞、子女分享家人動態 → 備「今日簡報」供聊天取用 → 主動開口「今天變冷囉、加件外套」。

## 7.5 延遲預算〔換 Gemini 後重估·待實測〕
開口 <1.5 秒（Gemini 出音 + Ditto 生臉串流 + WebRTC）。深功夫背景做、對話當下只取用＝深而不慢。

# 第 8 章 · 行為科學層（Elfie 偷骨丟皮）

> 偷行為科學、丟遊戲外觀。守健康照護底線、不過度遊戲化。

- **核心轉化**：賺幣 →「安心存摺」（不換折扣、靠親情驅動）。親情鼓勵閉環：完成任務 → 寧寧鼓勵 → 同步家人 → 家人按讚 → 寧寧轉達。
- **倫理化回饋**：獎勵「有沒有誠實記錄/完成」、不獎勵「數據好壞」（血壓高也肯定）。
- **去 streak 焦慮**：只正向累積、斷掉不懲罰、不顯紅字。
- **雙端分流**：本人端極簡，詳細趨勢與關懷摘要留給家人/照護者視角。
- **任務＝家人叮嚀不是作業清單**：對話式關懷口吻、不要 to-do 壓力。
- **雙用途**：同一套「今天要做的事」＝鼓勵 + 安全網偵測底子。
- 家人傳話層：寧寧當中間傳話人（語音↔文字轉換）、走雲端、刻意不做成微信群。
- Apple 健康/Watch 串接：零手動輸入、跌倒偵測接守護腦。
- 家庭挑戰：自由邀請、動態難度（按能力）、成就徽章、失敗不懲罰、合作不競爭。

| 檔 | 月費 | 每月送點 | 定位 |

## 9.2 ⚠️ 成本基礎待重算〔待 CFO/Codex 校〕

# 第 10 章 · 法規紅線（非醫療定性 · 守住）

> **產品定性鎖定**（Edward 6/27 拍板）：沐寧＝智慧健康/智慧照護（陪伴），**非醫療軟體**。免醫材查登、健康數字非特種個資——前提＝守住「非醫療」。

- **用藥＝鬧鐘**，不給個別用藥建議（「記得吃藥」OK、「以免血糖失控」不行）。
- **情緒陪伴不稱「諮商/治療」**（心理師法）。
- **危機＝轉介者**（1925/119/通知家人）、不自己扛、不說「沒事啦」也不當醫生。
- **❌ 文案禁用詞**：監測/監控、管理/控制血糖血壓病情、診斷/判斷/評估、治療/改善/緩解、追蹤病情、健康數據分析、諮商/心理治療、降低/預防某病風險、替代看醫生/不用去醫院（最危險）。
- **✅ 安全講法**：提醒吃藥（鬧鐘不接療效因果）、天冷多穿（不判讀數值）、提醒回診（反向強化、不取代醫療）。
- **防禦聲明**（App/著陸頁 footer）：「沐寧是陪伴與生活提醒的夥伴，不提供醫療診斷、治療或建議。健康問題請諮詢專業醫師。緊急狀況請撥 119／1925。」
- **隱私**：加密存好 + 條款揭露即接受 baseline；必須誠實揭露「聊天語音經第三方伺服器處理」（知情同意）。
- **點數定性**（Edward 6/27 拍板）：＝遊戲鑽石/服務預付、V1 不做信託履約。
- 上線前沙利曼 Gate 5 醫療宣稱掃描必跑。
