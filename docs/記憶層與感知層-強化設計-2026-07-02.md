# 一、記憶層（讓寧寧「真的記得長輩」）

## 借鏡誰（2026 SOTA）
- **Mem0**：消費級 App 首選；向量抽取＋計分檢索；**寫入不覆蓋、留多版本、檢索時定當下相關**；開源可自架。
- **Zep / Graphiti**：時序知識圖譜＋**事實有效期窗**（fact validity window）；適合「狀況會變」的場景。
- **Nomi AI（最貼近我們）**：**四層記憶 + Identity Core**（沉澱長輩核心自我、讀弦外之音、主動追問）。
- **Replika 2.0**：**記憶儀表板**讓用戶查看/修正記憶。

## 我們的設計（照 Edward 親述「四層記憶」· `AI設計期待` §1）
| 層 | 記什麼 | 壽命 |
|---|---|---|
| 🪨 核心檔案 | 你是誰（名字／家人／慢性病／老伴走了）| 永久 |
| 📖 長期回憶 | 重要事件／喜好（愛種花／碧潭釣魚）| 長期＋會合併 |
| 🗓️ 近況 | 最近在幹嘛／心情（迷韓劇／最近悶）| ~1–2 週滾動〔轉鈕〕|
| ☀️ 當天狀態 | 今天的世界（寒流／回診／新聞）| 每天刷新（見感知層每日排程）|

- **重要分數**＝情感重量 ＋ 健康（一律記＋記趨勢）＋ 重複出現（升級長期）＋ 一次性閒聊（快速淡掉）；低分不長存。
- **整理員**：定期合併重複、濃縮舊的、丟低分的——避免「只堆、自打嘴巴、記憶膨脹」。
- **底層**：Supabase + pgvector 語意檢索（見上「修正」）；`memory_items` schema 已有 `valid_from/valid_until/supersedes_memory_id`，支援「用新蓋舊、不抱過時的你」。

## 三個一定要補的機制（現在缺）
- **語意召回**：能把「幾個月前提過一次的細節」找回來（純計分不夠）。
- **時序 + 衝突**：每筆記憶帶「有效期/時間」；狀況變了（搬家/住院/過世）**保留多版本、回話時用當下版本**，不「自信說錯」。
- **遺忘衰減**：低重要性/過期的 temporary_event 自動淡出，維持召回品質與成本。

## 收斂雙軌（先解最痛）
- 現況兩套：`user_profile.json`（舊·中文 key·手工）vs `memory_items.json`（新·schema·AI 萃取）。
- 做法：**以 memory_items 為唯一真相**，把 user_profile 的「回憶/喜好」一次性搬進 memory_items + Identity Core；`chat_engine` 改讀同一套；舊檔退役。

## 子女信任（對「子女買」）
- **記憶儀表板**（抄 Replika）：子女可看/修正長輩的記憶——但**受同意分層 RLS 控管**（見子女授權設計），原始私密對話不攤開。

## 搭建步驟
1. 收斂成一套（user_profile → memory_items + Identity Core）＋驗證聊完真的寫回。
2. 底層接 Mem0：語意召回 → 事實有效期 → 遺忘衰減（照此序）。
3. 管家腦 post-turn 主動追問回路。
4. 子女記憶儀表板（接同意分層）。

# 二、感知層（讓寧寧「懂長輩此刻的處境」）

## 借鏡誰（2026 SOTA）
- **工具呼叫 / MCP grounding**：一個標準接口掛多個「即時事實來源」；**Google Maps Grounding（地點/天氣/路程）**、Search Grounding。
- **核心原則**：**精準、有範圍地投餵**（只在需要時給最小必要事實，不爆 token）。

## 我們的設計（把已設計的 7 感知源＋每日排程落成真的工具）
權威設計（`AI-SERVICE-DESIGN-v1` §Perception）已定 **7 種感知源**＋11 主題領域＋分寸規則。現況多為 stub，要把它接成**真的工具呼叫**：

**7 種感知源（照設計）**
- **時間/時區**：早晚語氣、作息時窗、提醒時機。← 最簡單、先接。
- **天氣**：散步建議、寒流/高溫警示 → **中央氣象署 CWA API**（台灣權威）或 Maps 天氣。
- **在地/區域**：在地推薦、台灣語言文化 → **Google Maps Grounding（MCP）**。
- **行事曆/作息**：提醒、回診、check-in。
- **家人狀態**（我先前漏掉）：誰能通知、誰最近來過 → 讀家庭資料（接同意分層）。
- **當前檢索**：11 領域（書/旅遊/在地/運動/財經/影劇/音樂/美食/新聞/心靈…）需要時才查真實事實、不能瞎編。
- **興趣圖譜**（我先前漏掉）：這位長輩平常愛什麼 → 讀記憶層。

**每日感知排程（SPEC §7.4，我先前漏掉）**
每天清晨背景跑一次：天氣（寒流→添衣量血壓）、今天回診/用藥/重要日子、挑過在地新聞、子女分享的家人動態 → 備「**今日簡報**」供聊天取用 → **主動開口**「今天變冷囉，加件外套」。這是「主動關心」的引擎。

**分寸規則（照設計）**：閒聊＝事實可有可無；**推薦＝事實必須查證不可瞎編**；健康/財經/旅遊/天氣＝日期、地區、來源新鮮度要顧。

**智慧鏡頭（`AI-SERVICE-DESIGN` §Wisdom Lens，我先前漏掉）**：可用佛/道/基督/台灣諺語/人生經驗/反思提問當溫柔陪伴層——不捏造經文、不強加宗教、危機時守護腦可蓋過。

## 運作方式
1. 管家腦看 `topic_perception_plan` → 判斷這輪需要哪些源。
2. 只呼叫需要的工具（用 Gemini Live 現成的 function calling）。
3. 把結果**最小、scoped** 注入即時語音的當下脈絡（不整包塞）。
4. 結果快取進 `perception_snapshots`（帶 expiresAt），避免重複呼叫。

## 感知 × 安全 × 主動（打通三腦）
- **餵守護腦**：寒流/高溫/空品差 → 主動關懷提醒（「今天很冷，膝蓋要保暖喔」）。
- **餵主動開口**：感知 + 記憶 → 主動關心（下雨且長輩愛散步 → 「今天下雨，改在家做點伸展好不好？」）。

## 搭建步驟
1. 接「時間/日期」→ 立刻擺脫寫死。
2. 接「天氣」（CWA）→ 真天氣取代 hardcoded「轉涼有寒流」。
3. 接「在地/交通/就醫」（Maps grounding）。
4. 感知結果接進管家腦（主動關心）＋守護腦（安全預警）。

# 三、對話互動層（讓她像真人、會主動、跟著你變）
> 權威：`AI設計期待-記憶人格互動` §3–4、SPEC §7.3。這是「不像金魚腦機器人」的關鍵層。

## 活的側寫（自動建檔、不讓用戶填表 · 差異化賣點）
- 從聊天**自動認識你、寫側寫**；每週重讀本週記憶 → 更新一張「這個人現在是誰」（近況／心情走向／最在乎／最近迷什麼）。
- **興趣權重**：常提 +分、一陣子沒提退分、每週固定活動穩定疊高。
- **改正／反向機制**：說「不喜歡」→ 降權；前後矛盾（搬家取消）→ **新的蓋掉舊的**；直接糾正（女兒叫美玲不是美華）→ 改檔。**跟著你變、不抱過時的你。**

## 主動開口（命脈 · 聊聊頁中央凸鈕）
- 先開口關心你的狀態／一天／心情。
- **帶她自己的話題**（配你興趣的新發現：電影／書／活動）。
- **雙向**：會分享、會反問、會討論——不是單向客服。
- 料來自：記憶 ＋ 今日感知 ＋ 配興趣新發現 ＋ 狀態關心。主動但**有分寸、不煩**。

## 待 Edward 拍板的「旋鈕」（`AI設計期待` §7）
- 近況滾動窗口：**1 週 / 2 週**？
- 重要分數門檻：記太多會雜、太少會忘，抓哪？
- 整理員合併頻率：**每天 / 每週**？
- 主動開口頻率：一天幾次才關心、不擾民？
