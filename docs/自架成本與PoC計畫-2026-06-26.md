## 1. 2D 軌成本帳（現在就能建）

### 1.1 結論表

| 項目 | 數字 | 來源 |
|---|---|---|
| 用什麼卡 | **RTX 3070 / 4060 等級**（消費級、一張即可起步）| 霍爾盤點【定】Audio2Face RTX 3070+ 即時 |
| 一張卡同時跑幾路對話 | **30-60 路【估】**（見 §1.3 推導）| 我外推 |
| 每月機器費（自架一台）| **約 NT$3,000-5,000 /月**【估】| 見 §1.4 |
| 攤到每用戶 | **約 NT$3-8 /人/月**【估】| 見 §1.5 |
| 一次性成本（2D 角色美術 + rigging）| **約 NT$30,000-80,000**（1-2 隻角色）| 見 §1.6 |

### 1.2 技術前提（霍爾盤點【定】）

- 路線：**Live2D Cubism（2D rigging）+ NVIDIA Audio2Face（音訊→嘴形 blendshapes、2025 已開源）**。
- Audio2Face 即時門檻：**RTX 3070+**（霍爾【定】）。我們的目標卡綽綽有餘。
- 重點：2D 軌的運算大頭是「音訊→嘴形參數」這顆引擎。Live2D 的 rig 渲染本身極輕（CPU/瀏覽器都能跑、霍爾【定】120-185ms）。

### 1.3 一張卡能同時跑幾路？（並發推導 · 全【估】）

這是 2D 成本帳的核心假設，我把推導攤開讓 Edward 看：

- Audio2Face 在 RTX 3070+ 能「即時」跑單路。即時 = 處理速度 ≥ 播放速度（RTF < 1）。
- 一張 RTX 4060/4070（比 3070 強）跑單路 Audio2Face 時，GPU 利用率【估】只吃一小部分（這類 audio→blendshape 模型比影像 diffusion 輕一個量級）。
- 嘴形推理是「短突發」工作——長輩說一句話才算一次，不是 24h 連續算。配上架構前提「臉只在 App 開時跑」，同時真正在「講話那一瞬間」的人數遠少於「同時開著 App」的人數。
- 保守抓：**一張消費級卡同時服務 30-60 路 App-open 對話**【估】。

> ⚠️ 這個 30-60 是【估】、不是實測。但 2D 軌就算我估錯一半（只跑 15 路），每用戶成本也才翻倍到 NT$6-16，**仍遠在 NT$150 之內**——所以 2D 軌的結論對這個假設不敏感、很穩。

### 1.4 每月機器費

兩條路、Edward 二選一：

| 方式 | 月費【估】| 說明 |
|---|---|---|
| **A. 自架一台（買卡）** | 攤提後 **NT$3,000-5,000 /月** | RTX 4070 整機約 NT$50,000-70,000，3 年攤提 + 電費 + 頻寬。一次性買、長期最便宜。 |
| **B. 租雲端 GPU 常駐** | **NT$8,000-15,000 /月**【估】| RTX 4090 等級雲端常駐月租。彈性但長期較貴。 |

> 起步期用戶少時，2D 軌甚至可以「按需開機」進一步省——但這層優化第一版不必做。

### 1.5 攤到每用戶

以「自架一台、月費 NT$4,000、同時撐 30-60 路」推：

- 一台同時撐 30-60 路 App-open ≈ 服務 **數百到上千名註冊用戶**（因為不是所有人同時開 App）。
- 保守抓一台服務 500-1,300 名活躍付費用戶：**NT$4,000 ÷ 500~1,300 ≈ NT$3-8 /人/月**【估】。

> 這跟 Sophie 財務模型 §1B 裡「Avatar lipsync 自架 ~$2（約 NT$64）」的量級一致、甚至更低——**2D 軌的臉成本在財務模型裡幾乎可忽略。**

### 1.6 一次性成本（2D 角色美術）

| 項目 | 估價 | 說明 |
|---|---|---|
| 2D 角色設計 + 繪製（1 隻可愛動物/管家）| NT$15,000-40,000 /隻【估】| 外包插畫師行情 |
| Live2D Cubism rigging（拆件 + 嘴/眼/頭部參數）| NT$15,000-40,000 /隻【估】| Live2D rigger 行情 |

> 架構骨架 §2.5：2D 軌「一組」。一次性、做完不再花。Live2D Cubism SDK 在營收門檻內免費（霍爾【定】），無授權月費。

## 2. 擬真軌成本帳 + Ditto PoC 計畫

### 2.0 為什麼這軌必須先 PoC

霍爾盤點【定】+ Edward 更正講得很清楚：
- 擬真即時的「便宜硬體」這塊**還沒解**（死路 A 的牆 2026 六月仍在）。
- Ditto 是「為即時而設計」（motion-space diffusion + 蒸餾 + 串流原生）、**單卡可跑是論文宣稱**，但**RTX 4090 的真實 fps 沒人替我們實測過**。
- Edward 更正重點：前 5 次撞牆是「自家拼圖 + 硬接 + 沒像樣 GPU + 沒照正規做法」，**不是做不到**。所以 PoC 要用**正規生成法 + 像樣機器**，別預設失敗。

**所以擬真軌的所有成本數字都掛在 PoC 結果上。先講 PoC 計畫、再講「若過」的成本。**

---

### 2.1 Ditto PoC 計畫 ⭐

#### 在哪種卡測

| 選項 | 成本 | 我的建議 |
|---|---|---|
| **租雲端 RTX 4090**（如 Vast.ai / RunPod 等級）| 約 **US$0.4-0.7 /小時**【估】| ✅ **首選**——便宜、隨開隨關、代表「消費級單卡」這條最務實的路 |
| 租雲端 A100 80G | 約 US$1.5-2.5 /小時【估】| 備案——若 4090 fps 不夠、加測一輪看「貴一級的單卡」能不能救 |
| 買本機 4090 | 一次性 NT$50,000-70,000 | ❌ PoC 階段不必、過了再說 |

> 「正規做法 + 像樣機器」= 直接用 Ditto 官方 repo 的標準推理路徑、在 RTX 4090（24GB，業界主流消費級旗艦）上跑，不自己拼湊。

#### 測哪幾個指標 + 成功判準

| 指標 | 怎麼量 | 成功判準（PASS 門檻）| 為什麼這個門檻 |
|---|---|---|---|
| **fps（單路即時）** | 跑一段語音、量輸出影格率 | **>= 25 fps** | 低於 25 fps 人眼感覺卡、達不到「像真人」 |
| **首幀延遲（first-frame latency）** | 從送音訊到第一幀出來 | **<= 500ms** | 對標 Tavus 600ms 整鏈、render 這層要更低 |
| **單路 GPU 佔用** | 量單路吃多少 VRAM + 算力 | 記錄實值 | 決定「一張卡能同時跑幾路」= 成本命脈 |
| **多路並發上限** | 逐步加路數到掉幀 | 記錄一張卡撐幾路 | **直接換算每用戶成本** |
| **畫質（主觀）** | 截圖/錄影給 Edward + 女巫看 | 無恐怖谷、嘴型對得上 | 像不像競品、Edward 拍板 |

#### 成功 / 失敗判準（整體）

- **PASS**：單路 >= 25 fps + 首幀 <= 500ms + 一張 4090 能同時 >= 3 路 + 畫質無恐怖谷。→ 擬真軌單卡成立、進 §2.2 算成本。
- **邊緣**：fps 達標但一張卡只能跑 1-2 路。→ 成本偏高（見 §2.2）、Edward 評估值不值。
- **FAIL**：4090 跑不到 25 fps 或首幀爆 1 秒。→ 老實回報、不硬推。可加測 A100 一輪看貴一級單卡能否救；若仍不行 = 擬真即時這條路這個時間點不便宜、回 2D 軌 + 預錄後援。

#### 跑多久 + 估花多少錢

| 階段 | 工時【估】| GPU 租用時數 |
|---|---|---|
| 環境架設（裝 Ditto repo + 依賴 + 跑通 sample）| 半天-1 天 | 2-4 小時 |
| 單路 fps / 延遲 / VRAM 量測 | 半天 | 2-3 小時 |
| 多路並發壓測（加路數到掉幀）| 半天 | 2-4 小時 |
| 畫質樣本錄製給 Edward/女巫 | 數小時 | 1-2 小時 |
| **合計** | **約 2 天工時** | **約 8-16 GPU 小時** |

**估花費：**
- RTX 4090 雲端：8-16 小時 × US$0.4-0.7 = **US$3-11（約 NT$100-360）**【估】
- 加上失敗重試 buffer × 2-3：**抓 NT$300-1,000【估】**
- 若加測 A100 一輪：再加 **NT$300-1,300【估】**
- **PoC 總預算上限：NT$1,000-2,500**（含 A100 備案 + 重試 buffer）

> 這就是我要 Edward 拍板的那筆錢。相對「擬真軌能不能成立」這個商業問題、這筆 PoC 錢極小、值得花。

### 2.2 若 PoC 過：擬真軌每用戶成本

**全部掛在 PoC 的「一張卡能同時跑幾路」結果上。我給三種情境讓 Edward 看區間：**

#### 硬體層級 × 每路成本

| 硬體情境 | 機器月費【估】| 一張卡同時幾路【PoC 才知】| 每路月成本【估】|
|---|---|---|---|
| **單卡 RTX 4090（樂觀）** | 買卡攤提 NT$2,000-3,000 /月 | 假設 3-5 路 | **NT$400-1,000 /路** |
| **單卡 RTX 4090（邊緣）** | 同上 | 假設 1-2 路 | **NT$1,000-3,000 /路** |
| **單卡 A100 80G（雲端租）** | NT$35,000-55,000 /月【估】| 假設 4-8 路 | **NT$4,400-14,000 /路** |
| **多卡 LiveAvatar 14B（5×H800）** | NT$300,000+ /月【估】| 45fps 多路、霍爾【定】5×H800 | 破表、不算 |

> 「每路」= 同時在講話的並發。和 2D 軌同邏輯：因為臉只在 App 開時跑、且只在講話瞬間算，一「路」並發背後對應數十名註冊用戶。

#### 攤到每用戶（關鍵換算）

承架構前提（臉只在 App 開時跑），假設一「路」並發 ≈ 服務 10-25 名活躍用戶【估】：

| 情境 | 每路月成本 | 一路服務幾人 | **每用戶月成本** | vs NT$150 |
|---|---|---|---|---|
| 單卡 4090 樂觀（3-5 路）| NT$400-1,000 | 15-25 人 | **約 NT$30-65 /人** | 🟡 勉強站住 |
| 單卡 4090 邊緣（1-2 路）| NT$1,000-3,000 | 10-15 人 | **約 NT$70-200 /人** | 🔴 邊緣到破表 |
| 單卡 A100（4-8 路）| NT$4,400-14,000 | 15-25 人 | **約 NT$180-700 /人** | 🔴 破表 |

> **這就是為什麼 PoC 的 fps/並發數是生死線**：同樣是 RTX 4090，「同時 5 路」和「同時 1 路」差 5 倍成本、直接決定擬真軌站不站得住 NT$150。

#### 一次性成本（做 4 個擬真臉）

架構骨架 §2.5【定】：擬真提供 **4 個形象（2 男 2 女）**，但「背後管家腦同一顆、共用」——所以一次性成本只是「各做一張臉的素材」、不是 4 倍系統複雜度。

| 項目 | 估價【估】| 說明 |
|---|---|---|
| 4 個擬真臉素材（拍攝/生成 source + 調校）| NT$40,000-120,000 | 每張臉 NT$10,000-30,000、含 source 影像 + 模型調校 |
| 整合進對話 pipeline（一次性工）| 含在開發工時內 | 4 張臉共用同一引擎、只是換 source |

> Ditto 是「單照片/source 驅動」類路線（霍爾盤點【定】Ditto 為即時設計），4 張臉的增量成本主要是素材、不是重建系統。

### 3.2 把臉放進 Sophie 財務模型看全局

Sophie §1B【定】：90/10 混合每用戶月成本目標 **~NT$150**（其中 Avatar lipsync 估 ~$2 = NT$64 的位置）。

- **走 2D 軌**：臉只吃 NT$3-8、**遠低於 Sophie 預留的 NT$64**。→ 整體成本還有下修空間、毛利 65% 穩。
- **走擬真軌（單卡樂觀 NT$30-65）**：臉吃掉的比 Sophie 預留的多 NT$0-1 倍。→ 整體月成本可能從 NT$150 升到 NT$180-215、**毛利從 65% 掉到 55-60%**【估】。還能玩、但要 Edward 接受。
- **走擬真軌（邊緣/A100）**：臉破 NT$100、**整體月成本破 NT$250、毛利崩**。→ 只能當「NT$899 深度守護方案專屬」、不進 NT$499 主力。

### 3.3 卡西法的技術判斷（結論 + 理由 + 替代方案）

**結論：分階段、別一次撲擬真。**

1. **2D 軌 = 主力保底、現在就建。** 成本趨近零、避開所有撞過的牆（死路 A 的 GPU 牆 + 死路 E 的 viseme 整圖切換牆）、毛利完全撐得住。**技術上沒問題、現在就能動。**

2. **擬真軌 = 先花 NT$1,000-2,500 跑 Ditto PoC、再決定。** 這是擬真能不能成立的唯一生死關。
   - PoC 過（單卡多路）→ 擬真當「進階方案」、NT$30-65/人、毛利掉一點但能接受。
