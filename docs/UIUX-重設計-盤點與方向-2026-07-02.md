# 沐寧 App · UIUX 誠實盤點與重設計方向（2026-07-02）

> 背景：Edward 2026-07-02「目前 app 的設計我不是很滿意」→ 女巫逐檔盤點 web/（index/styles/family/onboarding/landing＋資產）＋ 2026 對標調研。
> 本檔＝①為什麼不滿意（有憑據）②三個重設計方向（供 Edward 挑）。**未動任何程式**。

## 一、為什麼會不滿意（依嚴重度、全部有出處）

1. **🔴 滿版假資料感（致命）**：全 App 用了 **20 張西方人素材照**（randomuser.me）當阿嬤、美華、志明——12 歲的小寶用的還是一張**成年男性照**；每屏畫著假 iPhone 狀態列（9:41＋瀏海）。→ 打開永遠像「樣品屋海報」，不像產品。
2. **🔴 品牌人格分裂（致命）**：一個產品三套視覺——App 是 teal 綠的健康儀表板模板（styles.css 自己註記「往 Careon 調性」＝**從舊模板沿襲、不是為沐寧長的**）；門面 landing 卻是暖紙文學風（Noto Serif，反而貼 Edward 的品味）；寧寧本人插畫版/擬真版兩種畫風並存。Logo（暖橘水滴）的 DNA 在 App 裡完全沒延伸——**拿掉 logo 換任何名字都成立**。
3. **🟠 繁中長輩 App 塞英文裝飾標**：Today/Family/Together/Everyone/Achievements… 12px 加寬距英文小標——對 72 歲是噪音、對子女是「模板味」最大破綻。
4. **🟠 違反自家銀髮鐵律**：規劃明定「大字 24px+／按鈕 60px+」，實況有 10-11px 小字、26px 小按鈕——**阿嬤實際用不了家人分頁**。聊聊屏是唯一及格的（66px 結束鈕）。
5. **🟠 首頁背叛核心賭注**：命脈是「第一畫面就是她的臉、點一下就講」，實況首頁是 7 個模組疊的儀表板、寧寧縮在第 4 張卡 108×132px。**產品靈魂在畫面權重上輸給數據格。**
6. **🟡 溫度只在文案不在視覺**：文案很暖、容器全是冷的（漸層 KPI 卡、進度條）；心情天氣卡要的暖色語彙現系統接不住。
7. **🟡 圖示雙軌打架**：線稿 icon × 生 emoji 🏅👍 並存；同一顆愛心圖示身兼血壓/心情/關懷 8 處。
8. **🟡 family.html 過期**：仍是「子女端」視角＋把點數販售 UI 貼在關懷頁上——與「單一全家 App」定調矛盾（該收進設定/付費）。

**2026 對標一句話**：好的家庭照護 App 在走「冷靜大留白＋角色帶溫度（Finch 模式）＋數據穿上家的衣服」；Apple 健康的冷正是我們該差異化的地方。

## 二、三個重設計方向（供 Edward 挑）

| | A 溫暖日光 ⭐女巫推薦 | B 清潤呼吸 | C 沐寧童話 |
|---|---|---|---|
| 一句話 | 打開像翻家裡的相簿手帳：紙感、暖光、插畫家人 | 日系溫泉旅館的安靜：超大字大留白、每屏最多兩件事 | Finch 路線全押：App 就是寧寧住的家、家人頁＝全家花園 |
| 色盤 | 米黃底＋**暖橘琥珀主色**（呼應 logo 水滴）、teal 降輔助 | 保留 teal 但去漸層去重影、全平面 | 暖燈黃底＋暮色 teal＋燈橘 |
| 字體 | 標題 Noto Serif TC（跟門面合體、貼 Edward 已知品味）＋內文 17px+ | 純 Noto Sans 全字級加大 20-30% | 圓體標題＋Noto Sans 內文 |
| 插畫 | 柔水彩插畫統一全 App（家人頭像/徽章/寧寧，生圖之手可產） | 極少插畫、全押擬真聊聊臉 | 全插畫世界觀（寧寧的房間/全家花園/繪本任務） |
| 修掉哪些 | 1・2・3・6，直接為心情天氣卡鋪路 | 4・5・6 的密度面，最快有高級感 | 7＋識別度天花板、孫子最愛、為遊戲層鋪軌 |
| 工作量 | 中（重點在插畫資產包） | 低-中（幾乎純樣式） | 高（可分期） |
| 風險 | 低（與 logo/門面/roadmap 最對齊） | 最不獨特、識別靠文案撐 | 最貴 |

**三個方向共通的第一刀（不用等拍板就該砍）**：清掉 20 張西方素材臉、假 iPhone 狀態列、英文裝飾標。

**女巫推薦 A 的理由**：門面（landing）本來就是這個調性——**是 App 自己偏離了自家門面**；暖橘琥珀直接延伸 logo；Serif 貼 Edward 在 BeyondPath 已驗證的偏好；心情天氣卡、全家記錄簿都天然長在這個語彙上。C 可當 A 的第二期（全家花園/遊戲層先用 A 的插畫語彙起步）。

## 三、Edward 拍板（2026-07-02）：設計錨＝ **munea.net 正式官網**

Edward 裁示：對焦 **https://munea.net/ 的 Logo 與網頁用色/設計**；要求特質＝**溫暖、照護、設計巧思、有趣互動、中高齡標題、內文閱讀舒適、排版有美學感**。

### 3.1 官網設計基因（已實抓 warm.css 取證）
| 元素 | 值 | 用法 |
|---|---|---|
| 奶油底 | `#FAF7F2`（cream）/ 白卡 `#FFFFFF` / 薄荷 `#E8F2EE` | 頁底暖、卡片白、區塊薄荷 |
| 湖水綠（主） | `#3AA8A0` / 深 `#2E8A83` / 更深 `#236C66` | 主行動、重點 |
| **珊瑚橘（暖點綴）** | `#F4A261` / 深 `#E08B45` / 柔底 `#FBE7D2` | 溫度層：強調、徽章、心情暖色帶 |
| 墨色/次要/線 | `#33403D` / `#6B7B76` / `#E7E0D6` | 文字三階、髮絲線 |
| 字體 | 標題時刻 **Noto Serif TC**；內文 Noto Sans TC（Poppins 只當英文數字輔助） | 中高齡標題有分量、內文舒適 |
| 形狀 | 膠囊鈕（100px）、卡 14-20px 圓角、柔和無重影 | 親切不銳利 |
| 巧思 | 「陳奶奶的一天」時間軸敘事、照護閉環圓圖、可展開人物卡 | 有趣互動的 DNA |

### 3.2 App 設計系統定案（v1 token）
- **色**：頁底 cream；主行動湖水綠；**溫度層一律珊瑚橘系**（心情天氣暖帶＝coral-soft→coral、悶天＝灰藍不刺激、**不用紅標壞日**）；danger 只留安全場景。
- **字**：頁面大標/儀式時刻 Noto Serif TC ≥22px；內文 Noto Sans TC ≥17px、行高 1.7；**英文裝飾標全刪**、數字可 Poppins。
- **可及性鐵律**：內文對比 ≥4.5:1、點擊目標 ≥44px、長輩主要動線鈕 ≥60px。
- **元件**：膠囊主鈕、白卡＋髮絲線（少漸層少重影）、話題/心情用「郵票/貼紙」造型（巧思層）。
- **心情圖譜（Edward 2026-07-02 三輪迭代定案）**：
  1. **不用天氣圖示**（會跟 App 內「真天氣」撞衫、像天氣預報）→ 心情用自己的語言：**六格心情圖譜＝表情＋專屬色**：開心（珊瑚橘 #FBE7D2/#C25716）、愉快（杏黃 #F6ECD4/#9A6E14）、平穩（湖水綠 #E8F2EE/#1E7169）、疲累（灰綠 #EEEFEA/#5F6A61）、低落（灰藍 #E4EBF3/#3F5F80）、**煩躁（梅紫 #ECE1F0/#6E4488——刻意離開暖橘家族、跟開心一眼分開；不用警報紅）**。表情載意、顏色助讀。
  2. **一天≠只有一種心情**：日圖示＝當天總結（主要/最近），**當天有多種心情時角落加小圓點**；點該日展開「每次聊天各自的心情＋一句觀察」（早上煩躁/傍晚開心並列）。
  3. **版位**：個人卡**只顯示「今天」**（今天總結＋觀察句＋話題＋「看一週趨勢 ›」）；**週/月趨勢放獨立圖表頁**（家人可切換 週/月、點日展開）。
  4. 文案永遠是觀察不是判定（「有點火氣」而非「憤怒」；「比較安靜」而非「低落警示」）。
  - 引擎對應：情緒判讀由 1-5 級升級為**六類歸類**（每次聊天一筆、當天可多筆），趨勢庫存每筆、日總結顯示層計算。
- **圖像**：**清光 20 張西方素材照** → 家人頭像用暖插畫肖像（生圖之手產）；寧寧統一插畫版畫風；假 iOS 狀態列刪除。
- **互動巧思**（對齊官網 DNA）：首頁以寧寧為主角（臉/場景 hero＋一句話開聊）；「今天」用時間軸敘事感；全家健康圈用圓桌構圖呼應「照護閉環」。

> 定位一句話：**App 的湖水綠方向本來就對（跟官網同源），錯在只學了色、沒學到暖（珊瑚橘層）、巧思（敘事互動）與長輩閱讀舒適**。本定案＝把官網的完整語彙搬進 App。
