## 0. 先看這張表（你只要記這個）
| 跑法 | 改什麼 | 量什麼 | 為什麼 |
|---|---|---|---|
| 第一跑 · offline 調參 | max_size 1920→1280、steps 50→25 | offline it/s | 先看「畫小一點、步數減半」能拉多快、臉有沒有崩 |
| 第二跑 · online 即時 | 換 online cfg | **online it/s ← 這個才算數** | offline 是「算完整段」，online 是「邊收音邊吐畫面」＝產品真實情境 |

判準（細節在第 6 節）
- online it/s ≥ 25 且對嘴 OK → 🟢 翻案，4090 就能做即時數位人
- online it/s 18–24 → 🟡 4090 差一點，上 5090 應該過
- online it/s < 18 → 🔴 退「混合方案」（不追純即時）
> it/s 約略當 fps 看。25 it/s ≈ 25fps ≈ 人眼順。

## ⚠️ 全篇最可能卡的一步，先講
max_size 和 sampling_timesteps 到底是 inference.py 的指令參數（CLI flag）還是寫死在 cfg pkl 裡？Ditto 有的版本是 CLI、有的藏 cfg。所以第 2 節第一件事，就是教你用一行指令自己問出答案（30 秒），再決定走 A 路（直接下參數）或 B 路（改 cfg）。照做不會賭。

## 1. 開機後先回到「可跑狀態」（約 10–15 分鐘 / ~US$0.1–0.2）
> RunPod 的 pip 套件裝在系統區、pod 一旦 Stop/restart 就掉；但模型在 /workspace（持久碟）還在、不用重抓。所以開機只需「重裝兩個套件 + 確認模型還在」。

1-1 開 pod（範本＝之前那個 Better PyTorch CUDA 12.1）→ 進 Web Terminal。
1-2 重裝會掉的兩個套件：
```
pip install tensorrt==8.6.1 --extra-index-url https://pypi.nvidia.com
pip install cuda-python==12.1.0
```
看到 `Successfully installed tensorrt-8.6.1` 即過。tensorrt 報找不到→確認有加 `--extra-index-url`；版本衝突→先 `pip uninstall tensorrt cuda-python -y` 再裝。
1-3 確認模型還在：`ls -lh /workspace/ditto-talkinghead/checkpoints/`（看到 .engine 等大檔）。不見了才要重抓（多 20–40 分）。
1-4 進專案目錄：
```
find /workspace -maxdepth 3 -iname "inference.py" -path "*itto*" 2>/dev/null
cd /workspace/ditto-talkinghead      # ← 換成 find 出來的真路徑
```
1-5 sanity check：
```
python -c "import torch, tensorrt; print('torch', torch.__version__, '| cuda', torch.cuda.is_available(), '| trt', tensorrt.__version__)"
```
看到 `torch 2.x | cuda True | trt 8.6.1` ＝ 回到今晚 16fps 那個狀態了。

## 2. 第一跑 · offline 調參
**2-0（最可能卡、30 秒先問）**：
```
python inference.py --help 2>&1 | head -60
grep -rn "max_size" inference.py core/ scripts/ 2>/dev/null | head
grep -rn "sampling_timesteps" inference.py core/ scripts/ 2>/dev/null | head
```
- **A 路**：--help 有 `--max_size`/`--sampling_timesteps`（或 `--n_steps`）→ CLI 參數、直接加，走 2-1A。
- **B 路**：--help 沒有、但 grep 在某 .py/cfg 找到 `max_size=1920`/`sampling_timesteps=50` 賦值 → 藏 cfg、要改檔，走 2-1B。
> 判斷：sampling_timesteps 很可能在 cfg(B)、max_size 較可能 CLI(A)——但以你 grep 到的為準、別信我這句。

**2-1A · 直接下 CLI 參數**（把今晚 16fps 那條成功指令貼回來、只改這兩個值最穩）：
```
python inference.py --cfg_pkl ./checkpoints/ditto_cfg/v0.4_hubert_cfg_trt.pkl \
  --source_path ./example/your_face.png --audio_path ./example/your_audio.wav \
  --output_path ./out/test_1280_step25.mp4 --max_size 1280 --sampling_timesteps 25
```
若 flag 名不叫 --sampling_timesteps 而叫 --n_steps/--steps → 用 --help 的真名。

**2-1B · 改 cfg pkl**（grep 顯示藏 cfg 才走）：
```
cp ./checkpoints/ditto_cfg/v0.4_hubert_cfg_trt.pkl ./checkpoints/ditto_cfg/v0.4_hubert_cfg_trt.BAK.pkl
python3 - <<'PY'
import pickle, pprint
p="./checkpoints/ditto_cfg/v0.4_hubert_cfg_trt.pkl"
cfg=pickle.load(open(p,"rb")); pprint.pprint(cfg)
PY
```
看清楚 key 在第幾層後改值存回：
```
python3 - <<'PY'
import pickle
p="./checkpoints/ditto_cfg/v0.4_hubert_cfg_trt.pkl"
cfg=pickle.load(open(p,"rb"))
def setkey(d,key,val):
    if isinstance(d,dict):
        for k,v in d.items():
            if k==key: d[k]=val; print("set",key,"->",val); return True
            if setkey(v,key,val): return True
    return False
setkey(cfg,"sampling_timesteps",25)
setkey(cfg,"max_size",1280)   # key 若叫別名(max_dim/resolution)改這行字串
pickle.dump(cfg,open(p,"wb")); print("saved.")
PY
```
改完跑壞→`cp ...BAK.pkl ...trt.pkl` 還原。不確定 key 名→把 pprint 輸出貼回給卡西法。然後用「不帶那兩參數」的原指令跑（值已寫進 cfg）。

**2-2 量 offline it/s**：跑時 tqdm 印 `it/s`（或 `s/it`、it/s=1÷它）。也可 `time python inference.py ...` 後 it/s≈(音檔秒×25)÷real秒。預期比 16 高（可能 25–40）。**但這是 offline、第二跑才算數、且要看臉。**

## 3. 第二跑 · online 即時模式（這跑才決定 go/no-go）
> offline＝一次餵整段算完才出；online＝模擬「邊收音邊吐畫面」串流＝產品真實負載。
**3-1 換 online cfg**（檔名多 `_online`）：
```
find /workspace -iname "v0.4_hubert_cfg_trt_online.pkl" 2>/dev/null
python inference.py --help 2>&1 | grep -iE "online|stream|mode|cfg"
```
- 可能 1（換 cfg 就好）：`--cfg_pkl` 指到 online 那顆 + 同 2-1 的 max_size/steps。
- 可能 2（要 `--online`/`--mode` 旗標）：照 --help 加。
> 卡點：以 --help 為準；報錯缺參數→貼回給卡西法。

**3-2 ⭐ 量 online it/s（最重要的數字）**：同上看 tqdm。把這數字記大字：**online it/s = ____**。online 通常比 offline 慢（串流 overhead）、正常、正是要量它。

## 4. 看臉（人眼驗收，數字快臉崩=沒用）
**4-1 抓 mp4 下來**：RunPod File Browser 右鍵 Download；或 `cd ./out && python -m http.server 8080` → RunPod Connect→Port 8080 下載。
**4-2 三項都要 OK**：① 閉口音（ㄅㄇㄆ）嘴有真閉？（最易崩）② 嘴型跟聲音同步無延遲？③ 臉部無鬼影/糊/閃。跟今晚 16fps(1920/step50) 並排比、退步可接受嗎？「快但醜到像壞掉機器人」=不可接受、寧可降速保畫質。

## 6. 判準表（照 online it/s + 對嘴對號）
| online it/s | 對嘴 | 結論 | 下一步 |
|---|---|---|---|
| ≥25 | OK | 🟢 翻案 | 4090 就能做即時、PoC 過 → 進下一階段(接全鏈、量端到端延遲) |
| ≥25 | 崩 | 🟡 速度夠畫質不行 | 步數往回加(25→30→35)找剛好不崩的最低步數、再看 it/s |
| 18–24 | OK | 🟡 4090 差一點 | 上 RTX 5090 重測(算力+30–50%、多半過 25)、先別退方案 |
| 18–24 | 崩 | 🟠 雙重不足 | 加步數＋換 5090、成本/可行性重評 |
| <18 | 任意 | 🔴 退混合 | 4090 純即時不可行 → ①半即時(預生常用回應+即時只微調嘴型)②雲端 A100/H100 批次 ③換更輕對嘴模型。trade-off 給霍爾+蘇菲拍板 |
> 門檻訂 25：低於 25 人眼覺得卡、長輩陪伴對「自然」敏感。

## 7. 花多久花多少
開機重裝 10–15min ~$0.2｜第一跑(含 grep) 15–25min ~$0.2｜第二跑 5–10min ~$0.1｜下載看臉 5–10min ~$0.1。**合計 ~40–60 分、~US$0.5–0.7（NT$16–22）**。卡 cfg(B 路)來回多 20–30min。
