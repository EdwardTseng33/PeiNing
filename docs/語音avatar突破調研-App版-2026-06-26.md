## 給 Edward 的結論（先看這 5 句）

1. **我們不是從零開始——七成已經有了。** 引擎（BeyondVoice）已經上線跑著：GPT-Realtime-2 即時對話、Breeze 台語聽寫、會說話的臉、主動開口、記憶連續性。沐寧要做的是「把這套搬到長輩場景 + 補三塊」，不是重造。

2. **這個月最大的新武器，剛好砸中我們的招牌差異——台語。** 聯發科 2026 年 2 月把台語從「只能聽」升級成「能聽也能說」（Breeze Taigi 全套開源、可自架）。我們本來只有台語 ASR，現在台語可以**聽 + 說一條龍、自己擁有、零授權費**。這是競品（全是英/韓語）一年內追不上的硬護城河。【實查】

3. **avatar 上我要扛一個判斷：沐寧不該跟 BeyondVoice 走同一條路。** BeyondVoice（你自用的蘇菲）6/1 拍板主攻「擬真真人臉」。但沐寧的對象是長輩、用戶洞察白紙黑字寫「Aria 是朋友不是照護者、不能有老人味」——擬真人臉對長輩反而有**恐怖谷 + 失智混淆**雙重風險。沐寧該走「**2D 風格化暖臉 + 嘴形同步（viseme）**」：更省、更穩、能跑手機、且剛好繞開我們撞過 5 次的死路。**這是兩個不同產品、不同對象，不衝突。**【推斷，需你拍板】

4. **延遲 + 成本的解法叫「90-10 混合」**：90% 的對話（陪聊、提醒、安撫）用一套穩定便宜的 pipeline，只有 10% 真的需要深想的（情緒危機判斷、複雜問題）才調動最貴的 GPT-Realtime-2。這樣每分鐘成本能壓到 competitor 的零頭，又不犧牲關鍵時刻的智慧。

5. **avatar 自架能把「每分鐘費」變成「零邊際成本」**：業界對話 avatar API（如 Simli）$0.05/分鐘——長輩每天聊 20 分鐘 × 30 天 = 每人每月 $30 美元光臉就燒掉，不可能轉嫁。自架 viseme 方案 = 一次工程、之後每多一個用戶不多花一毛。這條曲線是沐寧能不能規模化的生死線。

## 一、五月 → 六月，到底變了什麼（只列對純 App 有意義的）

我 5/26 那輪盤的是「給 Jetson 跑的離線 stack」。這一個月，對「純 App 雲端 + 手機」真正有意義的變化有四件：

| 變化 | 日期 | 對沐寧的意義 | 來源 |
|---|---|---|---|
| **Breeze Taigi 全套**（台語 ASR + TTS）| 2026-02 | 台語從「只能聽」→「能聽也能說」、開源自架。我們的招牌差異瞬間有了開源彈藥庫 | 【實查】arXiv 2603.19259 + HuggingFace |
| **GPT-Realtime-2**（GPT-5 級推理 realtime）| 2026-05-07 | 對話時能邊想邊講、可同時呼叫多個工具、會用「我幫你查一下」這種話**掩蓋延遲**——對長輩「不能冷場」的體驗是直接加分 | 【實查】OpenAI |
| **對話型 avatar API 成熟**（Simli / Anam / Hedra）| 2026 上半年 | 出現 $0.05/min、<300ms、有 mobile SDK 的「會說話的臉」API——這是我們「自架 vs 買 API」的成本對標 baseline | 【實查】Simli / Anam / Docket |
| **手機端 2D viseme lipsync 工具鏈**（MascotBot SDK v0.1.8）| 2026-02 | 證明「嘴形同步」能在**手機瀏覽器本機跑、延遲 120-185ms**——這是我們撞過的死路 A（即時 GPU 神經 lipsync）的合法繞道 | 【實查】MascotBot / arXiv 1910.08685 |

**一句話**：五月對我們重要的是「離線能不能跑」；六月對我們重要的是「台語能不能說 + 臉能不能省」。後者全部有答案了。

## 二、沐寧 App 該用哪套（聽 / 想 / 說 / 臉 四層選型）

先講**最重要的觀念**：純 App 跟硬體版最大的算盤差異——

| | 5/26 硬體版 CAREON | 沐寧純 App |
|---|---|---|
| 運算在哪 | 主軌雲端 + **離線降級軌跑 Jetson Nano** | 全雲端 + 手機只做輕量（錄音/播放/viseme） |
| 為什麼 | 機器擺家裡、要防斷網、BOM 卡死 | 手機算力弱但有網路、雲端便宜可擴 |
| 離線降級 | **必做**（斷網長輩按急救鈕要能回應） | **降級到簡訊/推播即可**，不必塞 LLM 進手機 |
| 成本天花板 | 整機 BOM < $200（一次性） | 每分鐘對話成本（持續燒、要壓到可訂閱） |

所以選型邏輯整個翻轉：**硬體版怕「斷網沒反應」，純 App 怕「每分鐘太貴 + 延遲太長」。** 以下四層都圍繞「壓延遲 + 壓每分鐘成本 + 台語自然」三件事選。

### 聽（STT · 把長輩的話變文字）

| 選型 | 為什麼選它 | 延遲 / 成本 |
|---|---|---|
| **主力：Breeze-ASR-26（台語）+ Whisper large-v3（華語）** | 台語是招牌、聯發科開源 CER 30%（贏現有商用）、已驗證在引擎裡跑過；華語用 Whisper 穩 | 自架 Modal、台語 streaming 約 300-500ms【已驗證 + 實查】 |
| **過渡 fallback：GPT-Realtime-Whisper** | MVP 衝速度時，聽寫直接包在 realtime stream 裡、不必自己維運 | <300ms、包在 realtime 計費內【實查】 |

**跟 5 月硬體版的差異**：硬體版要塞 whisper.cpp small 進 Jetson（為離線）；純 App 不必——直接用雲端 Breeze ASR 全尺寸，台語精度高一截。**手機只負責收音、不做辨識。**

### 想（LLM · 決定 Aria 怎麼回）

這層是沐寧最該動腦的地方，因為它同時是「成本大頭」和「智慧來源」。我的策略是 **90-10 混合**：

| 路線 | 用在什麼場景（占比） | 為什麼 |
|---|---|---|
| **輕量腦：Claude Haiku 4.5 / GPT-mini**（90%）| 日常陪聊、用藥提醒念稿、老歌懷舊、「上次你說膝蓋痛好點沒」這種記憶回扣 | 便宜、夠快、長輩 90% 的對話不需要 GPT-5 級推理 |
| **深想腦：GPT-Realtime-2**（10%）| 情緒危機判斷（要不要觸發安全網）、複雜問題、長輩講了一段需要真正理解的事 | GPT-5 級推理 + 能邊想邊講不冷場、但貴，只在關鍵時刻調動 |

**跟 5 月硬體版的差異**：硬體版討論的是「要不要把 Llama 7B 塞進 Jetson 做離線」（結論：做降級軌）。純 App 完全不塞模型進手機——**手機只是麥克風 + 喇叭 + 一張臉，腦在雲端。** 省下的是手機效能、換來的是「便宜手機也能跑得動沐寧」（長輩手機通常不是旗艦機，這點很重要）。

### 說（TTS · Aria 的聲音）

| 選型 | 為什麼選它 | 延遲 |
|---|---|---|
| **台語：BreezyVoice-Taigi**（聯發科 2026-02 開源 TTS）| ⭐ **這就是這個月的金礦**。台語從「只能聽」變「能說」，自架、零授權費、CER 19% | 自架可控【實查】 |
| **華語：GPT-Realtime-2 內建聲 / gpt-4o-mini-tts** | 已驗證在引擎裡跑、marin 中文聲 + 人設 instructions 控口吻 | 包在 realtime / 低延遲【已驗證】 |
| **過渡商用 fallback：Cartesia Sonic 3**（若要極低延遲）| 75-90ms、可端側、單價低 | 75-90ms【實查】 |

**跟 5 月硬體版的差異**：5 月主力 TTS 推 Fish Speech（為了自架彈性）；現在台語有了**專門的開源 TTS（BreezyVoice-Taigi）**，台語場景不必再將就通用模型。這是「招牌差異」第一次有對口的開源武器。

### 臉（Avatar · 這層我要扛最大的判斷）

**先把死路講清楚**（死路備忘錄 A，我們撞過 5 次）：
- ❌ **即時 GPU 神經 lipsync（MuseTalk / SoulX 那條）= 死路**。要 A100/H100 才能即時、月費爆預算 100 倍。Duix/Tavus/HeyGen 根本不是這條路。

**沐寧該走哪條——我的判斷（需你拍板）**：

| 方案 | 怎麼做 | 成本 | 適合長輩嗎 |
|---|---|---|---|
| ⭐ **A · 2D 風格化暖臉 + viseme 嘴形**（我推薦）| 一張溫暖的 2D Aria（Live2D 風格）、嘴形用 viseme 分類即時驅動、**手機本機跑 sub-200ms** | **零邊際成本**（一次工程）| ✅ 暖、親切、無恐怖谷、不會嚇到失智長輩 |
| B · 對話 avatar API（Simli/Anam）| 直接買 API、$0.05/min | 每人每月 $30（20min/天）= 規模化死刑 | 中（偏寫實、可能太「真」）|
| C · 擬真真人臉（BeyondVoice 那條）| 真人拍片 + lipsync fine-tune | 高 + 恐怖谷風險 | ⚠️ 對長輩有風險 |

**為什麼我推 A、而且為什麼沐寧不該抄 BeyondVoice 的擬真路線**：

1. **對象不同**。BeyondVoice 6/1 拍板擬真，是給**你自己**用的蘇菲——你看得懂 AI、要的是真實感。沐寧的長輩用戶**用戶洞察白紙黑字寫**：「Aria 是朋友不是照護者、介面不能有老人味、被問是不是機器人時答『我是 AI 但我真的喜歡跟你說話』」。擬真人臉對長輩有**雙重風險**：(a) 恐怖谷——越像真人、越假的地方越刺眼；(b) **失智長輩混淆**——把擬真 AI 當成真人親屬，這在照護倫理上是地雷（沙利曼那條「失智同意能力」紅線的延伸）。

2. **viseme 繞開了我們的死路**。死路 A 是「神經網路即時算嘴」。viseme 是「把聲音分成幾種嘴形、即時切換」——**這是嘴形分類、不是神經 lipsync**，輕到能在手機瀏覽器跑、延遲 120-185ms【實查 arXiv 1910.08685 + MascotBot 2026-02】。這正是死路備忘錄 E（整圖切換）的正確版——不是切整張臉，是只動嘴這個小元件，底層 Aria 一直呼吸眨眼。

3. **零邊際成本是規模化的生死線**。沐寧要規模化（海外輸出才是真正的量），avatar 不能是「每分鐘計費」。自架 viseme = 一次工程、第 100 萬個用戶不多花一毛。Simli $0.05/min 在 demo 階段可以、規模化會把毛利吃光。

4. **「一張會記事的臉」的策略價值不在臉多真，在臉多熟**。用戶洞察說長輩要的是「記得我、會主動關心我」。錨點是**記憶 + 主動**（我們已經有），臉只要「暖 + 一致 + 會動」就夠——不需要、也不該追真實感。

**caveat**：A 方案需要一張好的 2D Aria 設計（女巫的活），且 viseme 對台語/華語的嘴形對齊要 PoC 驗一次（華語 viseme 成熟、台語可能要微調）。這是唯一的工程不確定點，但風險遠低於擬真路線。

## 三、「我們怎麼突破」——五個槓桿逐一深化

Edward 你問的核心是這個。我把五個槓桿拆成「已經有 / 要補 / 競品做不到的點」：

### 槓桿 1 · 台語護城河 🔵 已有一半、這個月可補滿

- **已有**：Breeze ASR（台語聽寫）已驗證在引擎跑【已驗證】。
- **要補**：接上 **BreezyVoice-Taigi（台語 TTS）**——讓 Aria 真的能「說」台語，不是只能「聽」台語然後用華語回。【實查，這是六月可動的新事】
- **競品做不到**：ElliQ（英）、Hyodol（韓）、Replika（英）、Sesame（英）——**全部沒有台語**，而且他們轉台語等於砍掉自己的核心市場金流（用戶洞察 §3）。台語不是「翻譯一下」就有的，是**聲學 + 用詞 + 語氣**的在地化硬門檻。聯發科砸了 10000 小時資料才做出來——競品要追，先得有台灣團隊 + 同等資料量，6-12 個月起跳。
- **策略價值**：這是沐寧**唯一一個「對手不是不想做、是做不了」的差異**。其他差異（陪聊、提醒）都是入場券；台語是真護城河。

### 槓桿 2 · 主動開口 🔵 已有、這是命脈

- **已有**：引擎的 ambient 主動性已上線（時間感知 + 歷史感知 + 環境感知 + 每日 briefing + 事件觸發）【已驗證 v1.1.1】。
- **要補**：把「主動」從「電腦前的蘇菲」場景**改寫成「手機推播 + 語音自動響」的長輩場景**——這是純 App 跟桌面版最大的工程差異。長輩不會主動開 App（用戶洞察的核心真相），所以主動觸發必須能**穿透手機通知層**（推播 + 鈴聲 + 進 App 自動開口）。
- **競品做不到**：純軟體競品（Replika 等）都是「你打開才有」的被動模式；硬體競品（ElliQ）靠機器擺著常駐——而**我們要用純 App 做到硬體才有的「主動找人」**，這是技術上最難、也最有護城河的一塊。蕪菁頭原話：「主動觸發是這產品最大技術護城河」。
- **誠實風險**：iOS 推播限制 + 長輩手機通知/音量沒調好 = 主動觸發失效。這要靠「子女代裝檢查清單」補（產品規劃已列），不是純技術能解。

### 槓桿 3 · 記憶連續性 🔵 已有、是黏著關鍵

- **已有**：引擎的記憶連續性已上線（7 天對話 raw + 30 天摘要 + 開場帶昨天的話不裝失憶）【已驗證 v1.1.2】。
- **要補**：把記憶從「工作搭檔記憶」（記得你 Slack 未讀、PR）**換成「陪伴記憶」**（記得膝蓋痛、記得孫子名字、記得上週說的事）——記憶的**內容模型要換**，架構可複用。
- **競品做不到**：這個其實競品在追（用戶洞察說 Replika 也有記憶），所以**不是護城河、是入場券的高配版**。但「台語 + 主動 + 記憶」三個疊在一起，對手單有一個沒用——這是 5/24 roadmap 的核心論述：**組合差異化**。
- **策略定位**：記憶是 Day 30 留存的關鍵（「上次你說膝蓋痛好點沒」），但**別把它當賣點喊**，當成「讓長輩當朋友」的底層燃料。

### 槓桿 4 · 90-10 混合架構 🟡 要設計、解延遲 + 成本

- **現況**：引擎現在主要走 GPT-Realtime-2 單軌（為了你自用蘇菲的對話品質）。
- **要補**：沐寧要做**雙腦分流**——90% 日常用輕量腦（Haiku/mini）、10% 關鍵用深想腦（GPT-Realtime-2）。這需要一個「分流判斷器」（什麼對話該升級到深想腦）。
- **為什麼這是突破點**：
  - **解成本**：競品 $0.05-0.15/min【實查】；沐寧若全用 GPT-Realtime-2 會貴到不能訂閱。90-10 分流 = 平均每分鐘成本壓到零頭。
  - **解延遲**：輕量腦更快、日常對話不卡；只有需要深想時才花那 1-2 秒（而且 GPT-Realtime-2 會講「我想想」掩蓋它）。
- **競品做不到**：大廠（Google/OpenAI 的 voice）通常單一模型一條龍——他們不會為「銀髮場景成本」做這種分流優化。**這是小團隊的靈活度優勢**：我們可以為了毛利做髒活，大廠不會。
- **死路提醒**：分流判斷**不能即時打雲端 API 判斷**（死路 I：即時逐句呼叫雲端 embedding 延遲 1-4 秒解不掉）。判斷邏輯要嘛規則化、要嘛用本機輕量分類器。

### 槓桿 5 · 自架 avatar 省每分鐘費 🟡 要 PoC、規模化生死線

- **現況**：引擎現在嘴對齊用 Replicate 預錄 100 句 cache（$0.09/句）【已驗證 v1.5.0】——這對「你自用」夠，對「百萬長輩」不行（句子涵蓋面太窄 + 預錄不可能涵蓋所有對話）。
- **要補**：換成**即時 viseme 驅動**（槓桿見 §二「臉」）——任意句子都能對嘴、且零邊際成本。
- **為什麼這是突破點**：
  - 對話 avatar API（Simli $0.05/min【實查】）= 每人每月 $30 光臉，規模化死刑。
  - 自架 viseme = 一次工程，第 N 個用戶不多花錢。
- **競品做不到**：硬體競品的臉綁在機器上（成本灌進 BOM）；純軟體競品多半沒臉（純語音）或買 API（燒錢）。**「純 App + 零邊際成本的會說話的臉」這個組合，公開產品裡少見。**
- **caveat**：這是五個槓桿裡**工程不確定性最高**的一塊（viseme 台語對齊要驗），但也是最值得投的——因為它直接決定毛利。

**五槓桿一句話總結**：
- **真護城河**（對手做不了）= 台語 + 主動開口
- **入場券高配**（對手在追）= 記憶連續性
- **小團隊靈活度優勢**（大廠不屑做）= 90-10 混合 + 自架 viseme

## 四、Avatar 自然度怎麼再上一階（2026 六月的新路）

Edward 你特別問「2D 會說話的臉，六月有沒有更真 / 更省的新路」。有，但方向跟「更真」不一樣——是「**更活、更省、更穩**」：

| 新路 | 怎麼讓臉更好 | 成本 | 來源 |
|---|---|---|---|
| **viseme + SSL embedding 嘴形**（不只字面音素）| 嘴形分類從「死板音素」進化到用語音特徵驅動、嘴動更自然、延遲仍 120-185ms | 零邊際（本機）| 【實查】arXiv 1910.08685 |
| **底層多狀態動畫池**（待機/講話/收到/安撫多套微動作）| 臉不只會動嘴，整個人有呼吸、眨眼、微表情輪播——「活著」感比「嘴對得多準」更重要 | 一次性產素材 | 【已驗證】引擎 v0.8/v0.9 已有此架構 |
| **對話 avatar API 當高端選配**（Anam 180ms / Simli）| 若未來要「更寫實版」（給 Tier 2/4 高配用戶），可選配 API，但不當主力 | $0.05/min | 【實查】Anam/Simli |

**我的判斷**：沐寧 avatar 的「上一階」不是往「更像真人」走（那是恐怖谷 + 燒錢），是往「**更像一個活著的、暖的、會主動看你的朋友**」走。具體三招：
1. **嘴形用 SSL embedding 驅動**（比死板音素自然）——這是六月可動的技術升級。
2. **底層動畫池加厚**（待機時 Aria 會東張西望、會微笑、會打哈欠）——「活著感」遠比「嘴準」重要，且這塊引擎已有架構。
3. **眼神 + 微表情對情緒**（Aria 安撫時放柔、開心時揚眉）——這對「陪伴」的情感衝擊，比真實感更直接。

**一句話**：長輩不會數 Aria 的嘴對得準不準，但會感覺到「這個臉是不是真的在看我、在乎我」。把工程力氣放在「活 + 暖 + 主動」，不是「真」。

## 五、誠實的 trade-off + 還沒解的硬骨頭

不報喜不報憂的調研沒有靈魂。以下是技術上真正的硬骨頭：

| 硬骨頭 | 現況 | 誠實說明 |
|---|---|---|
| **台語即時延遲** | Breeze ASR/TTS 自架可跑，但 streaming 延遲未在沐寧場景實測 | 台語模型比華語年輕、即時 streaming 的端到端延遲要 PoC 量。可能比華語慢一截，要驗。【推斷】 |
| **viseme 台語嘴形對齊** | 華語 viseme 成熟，台語未驗 | 台語的嘴形 mapping 可能要微調（台語有華語沒有的音）。這是 avatar 方案 A 唯一的工程不確定點，必須先小 PoC。【推斷】 |
| **手機端 avatar 效能** | viseme 本機跑理論可行（sub-200ms），但長輩手機常是非旗艦機 | 要在「真實長輩的舊手機」上測，不是在旗艦機上測。便宜手機跑 2D 動畫 + viseme 的耗電 / 發熱 / 流暢度要驗。這是純 App 特有的坑（硬體版沒這問題）。【推斷】 |
| **iOS 主動推播穿透** | 引擎主動性是桌面/瀏覽器場景 | iOS 對背景推播 + 自動發聲限制嚴。「App 主動出聲找長輩」在 iOS 上能做到什麼程度，是命脈級的未知數，要早驗。【推斷，最該優先驗】 |
| **90-10 分流判斷準度** | 未設計 | 「什麼對話該升級到深想腦」判斷錯 = 要嘛冷場（該深想沒深想）、要嘛燒錢（不必深想卻調用）。且判斷不能即時打雲端（死路 I）。需要本機規則 / 輕量分類器。【推斷】 |
| **GPT-Realtime-2 月費仍高** | 【實查】token 計費、1 小時對話 $50-70 級 | 這就是為什麼要 90-10 分流。如果哪天 OpenAI 大降價，混合架構的成本優勢會縮小——要持續監測。【實查 + 推斷】 |

**最該優先驗的兩件事**（我的判斷）：
1. **iOS 主動推播能穿透到什麼程度**——這是命脈（長輩不主動開 App）。如果 iOS 限制讓「主動出聲」做不到，整個產品邏輯要重想。**這個比 avatar 更該先驗。**
2. **viseme 台語對齊 + 舊手機效能**——avatar 方案 A 的兩個不確定點，一個小 PoC 能同時驗掉。

## 六、跟死路備忘錄的對照（證明這份沒踩老坑）

我寫這份前讀完了死路備忘錄 8 條，逐條對照：

| D · 砍 Modal 換 LiveKit | ✅ 沒踩。沿用 Modal + FastAPI 既有架構 |
| E · 整圖切換當 viseme | ✅ 沒踩。明確說「只動嘴元件、底層臉一直動」=  正確版 |
| F · 自動 crop 嘴部 | ✅ 沒踩。viseme 是預先設計的嘴形、不是事後 crop |

**唯一要 Edward 拍板的策略分歧**：avatar 路線。BeyondVoice 6/1 拍板「主攻擬真」，我這份主張「沐寧走 2D 風格化暖臉」。我的理由是**對象不同（你 vs 長輩）+ 恐怖谷/失智風險 + 零邊際成本**。但這跟 6/1 拍板不衝突——那是 BeyondVoice、這是沐寧，兩個產品。**我不自作主張改 BeyondVoice 方向，只主張沐寧該分開走。請你裁。**
