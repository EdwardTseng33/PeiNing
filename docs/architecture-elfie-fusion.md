# 陪寧 PAINING × Elfie — 融合基礎架構設計

> 2026-06-26 · 把 Elfie 的遊戲化做法轉化、融入陪寧現有 5 分頁架構
> 視覺仍守 CAREON 療癒綠 + 長輩無障礙；不照搬 Elfie 的粉紅活潑與賺幣換折扣

---

## 設計原則
1. **不照搬「賺幣換折扣」** —— 長輩動機弱（不為換 Nike 而吃藥），轉化成「親情驅動的健康鼓勵」
2. **不另開新分頁** —— 全部長在現有 5 分頁（寧寧/狀態/聊聊/家人/設定）上
3. **守長輩底線** —— 點數低調、任務少（2-3 件）、主軸陪伴+健康、不過度遊戲化
4. **守 CAREON 視覺** —— 療癒綠穩重，不染 Elfie 的粉紅活潑

## 一、核心轉化：賺幣 →「安心存摺」
- 不叫幣、不換折扣，叫**安心存摺**（暫名，可再定）
- 長輩最強動機 = **被在乎的人看見**
- 完成健康任務 → 累積 + 寧寧即時鼓勵 → 傳到家人 → 家人鼓勵 → 回長輩

### 親情鼓勵循環（閉環）
```
寧寧提醒任務
   → 長輩完成（吃藥 / 量血壓 / 散步 / 陪寧寧聊天）
      → 累積 + 寧寧即時鼓勵
         → 同步到家人分頁：「媽媽這週很棒，連續吃藥 6 天」
            → 家人一鍵按讚 / 鼓勵
               → 寧寧轉達：「美華幫你按了讚，說你好棒」
                  → 長輩更有動力 → 回到頂端
```
比 Elfie「換品牌折扣」更貼長輩、更貼陪寧「家人替身」定位。

## 二、融入 5 分頁

| 分頁 | 原本 | 融入後長出 |
|---|---|---|
| **寧寧**（首頁）| 待機 + 今天的提醒 | 「今天的提醒」升級成「**今天一起完成**」：2-3 件任務卡，打勾 + 寧寧鼓勵 + 累積；頂部低調顯示「這週很棒」 |
| **狀態** | 身體數據四格 | 加「**目標 vs 達成**」（吃藥 2/3、散步 20/30 分）+ **安心存摺**（連續天數、本週成就、健康趨勢）|
| **聊聊** | 語音對話 | 陪寧寧聊天也算「完成一件事」，鼓勵長輩多開口（陪伴本身有回饋）|
| **家人** | 家人留言 | 加「**媽媽這週**」卡：家人看到長輩努力成果 + 一鍵鼓勵（親情循環的關鍵節點）|
| **設定** | 設定 | 安心存摺規則、要不要讓家人看到、提醒/任務管理 |

## 三、基礎架構（底層三塊資料 + 流程）

### 3.1 三塊核心資料
1. **任務 Task**：今天該做的健康事
   - 屬性：類型（吃藥 / 量測 / 活動 / 陪伴）、時間、完成狀態、對應鼓勵值
2. **安心存摺 Wallet**：累積與成就
   - 屬性：累積點、健康紀錄[]（血壓 / 睡眠 / 用藥 / 步數）、連續天數、本週/本月成就
3. **家人連結 Family**：親情層
   - 屬性：成員[]、留言[]、鼓勵（按讚）[]、可見範圍（長輩可決定讓家人看到什麼）

### 3.2 核心流程
- 任務由「管家腦」每早排定（感知 + 記憶）→ 寧寧提醒
- 長輩完成 → 寫入存摺 + 觸發寧寧鼓勵 + 同步家人
- 家人鼓勵 → 回寫 → 寧寧轉達

### 3.3 ⭐ 雙用途：任務系統 = 鼓勵 + 安全網
- **完成** → 正向：累積 + 鼓勵
- **長期未完成**（沒吃藥 / 沒互動 / 數據異常）→ **守護腦警訊** → 通知家人 / 1925 / 119
- 同一套「今天要做的事」資料，**一兼二用**：加遊戲化的同時，把安全網的偵測底子也打好了

## 四、守住的底線（不過頭）
- 點數低調（背景累積、不滿屏數字、不彈窗轟炸）
- 任務少（一天 2-3 件，不造成壓力）
- 主軸是陪伴 + 健康，鼓勵只是溫柔的副作用
- 法規不變：用藥=鬧鐘不給建議、陪伴不稱諮商、危機=轉介

## 五、落地（下一版做進畫面）
1. 首頁「今天的提醒」→「今天一起完成」任務卡（可打勾 + 完成回饋）
2. 狀態頁加「目標 vs 達成」+ 安心存摺區
3. 家人頁加「媽媽這週」+ 一鍵鼓勵
4. 一併處理：嚴格對齊 Claude Design 設計稿（圖示換線性、不用表情符號）+ App 圖示對齊標誌系統

## 六、行為科學守則（霍爾深度調研補強 · 2026-06-26）

> Elfie 教我們的是「怎麼用行為科學讓人持續顧健康」，不是「怎麼把健康做成遊戲」。**偷骨（行為科學），丟皮（遊戲外觀）。**

1. **倫理化回饋**：獎勵「有沒有誠實記錄／完成」，**不**獎勵「數據好壞」。長輩血壓高也一樣肯定（誠實量了就好）→ 避免罪惡感與藏數據。這是 Elfie 行為科學的精髓。
2. **去除 streak 焦慮**：只做正向累積（「這個月 23 天準時吃藥」），斷掉不懲罰、不顯紅字「中斷！」。長輩斷紀錄會羞愧到不敢再用，跟年輕人「明天補」完全不同。
3. **雙端分流**：長輩端極簡（血壓平穩 = 綠笑臉 + 白話）；詳細趨勢圖留給子女在家人端看。認知負荷上限長輩遠低於年輕人。
4. **一鍵回診摘要**：用藥順從 + 血壓趨勢 + 近期不適 → 一鍵生成給醫生看的摘要。長輩回診剛需 + 未來 B 端伏筆。放狀態頁／用藥窗。
5. **任務 = 家人叮嚀，不是作業清單**：用寧寧對話式關懷口吻（台語「中午的藥別忘了」），不要 to-do 打勾的壓力感。修正 §二「任務卡」：以對話叮嚀為主、溫和呈現為輔。
6. **戰略定調**：Elfie 最弱 = 家庭連結（半成品）；陪寧最強 = 長輩用 + 子女看雙邊。**不做「長輩版 Elfie」，做「Elfie 想做卻沒做成的家庭健康陪伴」**。家人分頁押重注。
7. **B 端伏筆**：累積「真的讓長輩準時吃藥」的數據，未來對保險／長照／藥局有價值（這正是 Elfie 的賺錢核心）。

**臨床佐證**（遊戲化真的有效）：Elfie 埃及同儕審查研究——服藥順從 +40%、血壓 155/96→121/80、一月留存 40.12%（健康 App 罕見的高）。

## 七、家人訊息怎麼傳（寧寧傳話架構 · 2026-06-26）

> 核心：**刻意不做成微信群**。家人訊息不是長輩自己打字傳，是「寧寧當中間傳話人」，訊息走雲端。

**兩個方向**
- **子女 → 長輩**：子女手機打字／錄音 → 存雲端「家庭群組」→ 推播長輩手機 → **寧寧台語念給長輩聽** + 家人頁顯示文字。長輩不用讀字、不用操作。
- **長輩 → 子女**：長輩跟寧寧說「跟美華說我想她」／按「請寧寧提醒打給我」→ **寧寧整理成訊息** → 存雲端 → 推播子女手機。長輩不用打字。

**架構三塊**
1. **家庭群組（Family Group）**：子女裝機時建立，把長輩＋多位子女綁成一個「家」。一對多。
2. **雲端信箱（Cloud Inbox）**：訊息存雲端（非手機對手機直傳）→ 長輩換手機／關機不丟、多子女收同一份、可回溯。
3. **寧寧傳話層（Butler Relay）**：長輩端「聽＋說」、子女端「打字＋看」，寧寧在中間做格式轉換（語音 ↔ 文字、整理語氣）。

**子女端**：子女在自己手機有陪寧的「家人視角入口」（看長輩狀態、收通知、傳話、視訊）。同一個雲端、兩種視角——長輩端是滿版陪伴，子女端是「照顧爸媽」。

**為什麼**：長輩最大障礙＝打字＋讀字，寧寧念／傳話繞開；走雲端＋推播＝存得住、App 關著也收得到（接推播層）。

**底層（給工程）**：雲端一張「家庭群組 + 訊息串」的表；子女送出 → 即時同步長輩端 + iOS 推播；訊息含類型（文字／語音／系統通知如按讚、回診提醒）。

## 八、健康數據來源：Apple 健康 / Apple Watch 串接（2026-06-26）

> 現況：狀態頁數據原為手動示意，無自動來源。本段補設計。

- **接 Apple 健康（HealthKit）**：長輩授權一次 → 步數、心率、睡眠、血壓自動進「狀態」頁與安心存摺，**長輩零手動輸入**（打中長輩最大障礙＝被動 + 不會操作）。
- **Apple Watch 殺手鐧**：跌倒偵測 + 心率異常 → 直接接「守護腦」安全網 → 自動通知子女 / 119。把被動的健康數據變成主動的安全偵測。
- **彈性分層**：有 Apple Watch → 自動抓 + 跌倒求救；無 → 手動 / 寧寧開口問 / 接藍牙血壓計。不強迫買錶。
- **擺放**：狀態頁數據標「來自 Apple 健康」；設定頁「連接 Apple Watch / 健康」。
- **價值**：長輩零負擔 + 安全網硬化（跌倒自動求救是長輩照護的核心賣點，也是子女付費動機）。

## 九、家庭健康互動：合作型遊戲化（2026-06-26）

> 現況：只有單向「按讚」+ 個人安心存摺，無家人互動遊戲。本段補設計。
> 核心：**Elfie 是「個人賺幣」，陪寧做成「家庭一起玩」——健康是藉口，家人互動才是目的。** 這是陪寧 vs Elfie 的差異化（家庭連結 = 陪寧最強）。

- **全家一起達標（合作不競爭）**：共同目標如「這週全家一起走 3 萬步」，阿嬤散步＋子女運動＋孫子跑跳都計入，湊同一個目標，達成全家慶祝。**絕不做 PK**——長輩輸了會挫折（接霍爾 §六「去 streak 焦慮」）。
- **隔代小互動**：阿嬤吃藥 → 孫子端收到「阿嬤今天很乖」→ 孫子回塗鴉／貼圖／語音 → 寧寧念給阿嬤聽。健康成為阿嬤 ↔ 孫子的互動契機（情感價值最高）。
- **家庭集章月曆**：全家集滿一個月 → 月底家庭聚餐。線上成就帶出線下團聚。
- **守長輩**：合作不競爭、長輩端輕量（就是「收到鼓勵 + 簡單回應」）、複雜遊戲機制放子女／孫子端（雙端分流）。
- **擺放**：家人頁加「家庭任務」卡；現有「按讚」升級成「互動回應」（塗鴉／貼圖／語音）。

## 十、家庭挑戰完整機制（2026-06-26 · Edward 拍板）

> 把「家庭挑戰」從「全家強制一個目標」做成完整玩法：**自由邀請、動態難度、成就徽章、記錄簿**。核心：自由參加、公平難度、成就紀念、不懲罰。

**1. 發起與邀請（不強制全員）**
- 任一家人發起挑戰（走路／早睡／量血壓／用藥全勤／喝水…）
- 發起時**勾選邀請對象**（自由選、不強制）；被邀者可接受／婉拒（婉拒無壓力，不影響感情）
- 湊齊接受者 → 挑戰開始

**2. 動態難度（依人數 + 能力，公平、長輩不當負擔）**
- 目標 = 各參加者「量身達標線」之和（非齊頭）
- 依能力加權：阿嬤 3,000 步、年輕人 8,000 步，各湊各的份 = 全家達成
- 人少也成立、人多更熱鬧；誰都不拖累誰

**3. 成功 → 家庭記錄簿 + 成就徽章**
- 達成 → 全家慶祝 + 記入**家庭記錄簿**
- 記錄簿（點開）：歷場挑戰歷史（誰參加／結果／日期）+ **每個人的徽章牆**
- 徽章類型：達成、連續、首次參加、貢獻王、**長輩專屬**（阿嬤達成自己份的特別徽章）、**隔代**（阿嬤+孫子同場）
- 記錄簿 = 家庭共同回憶（情感價值 > 成就數字）

**4. 失敗不懲罰**（守長輩 · 接 §六去 streak 焦慮）
- 沒達成不罵、記「參加過」、下次再來；重點是一起參與、不是輸贏

**5. 長輩友善**
- 阿嬤份額按能力（不被當負擔）；達成給專屬徽章 + 家人看到鼓勵
- 長輩端極簡：收邀請 → 一鍵接受 → 寧寧幫她記（步數／吃藥）；發起、難度設定等複雜功能放子女端
