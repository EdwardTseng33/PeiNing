## 1. 視覺體驗（女巫 · Gate 2）

**舒適 85、驚艷 55。** 不是普通健康 App、有真設計手法（三層次標題/卡片質感/漸層光暈/浮動 chip），但停在「體面的好設計」、差一口氣到「會被記住」。根因＝**克制過頭、缺靈魂注入**。

10 維最低三項：**創新/記憶點 4.5**、空狀態 5.5、資訊密度 6.5（首頁/狀態/家人/裝機內容只填視窗上 2/3、下方空蕩）。最強：一致性 9.0（封閉色票零違憲）、微文案 9.0（有溫度）。

動態：全 App 只 6 個 @keyframes、唯一有靈魂的 floaty（頭像浮動）只用在聊聊一處 → **三個日常頁是靜止的、不會呼吸**。

🔴 P0 補靈魂（70→85+）：
1. **給寧寧專屬的臉**（換掉 randomuser 西方照）→ 記憶點 4.5→8。女巫建議**親和擬人管家**方向、需 Edward 拍板、可先出風格稿。
2. **三日常頁會呼吸**：floaty 蔓延、金色獎章發光脈動、進度條流光（純 CSS、零風險）。
3. **任務完成喜悅時刻**：打勾彈跳＋漣漪＋存摺數字滾動＋寧寧點頭。
🟡 P1：填底部空蕩（首頁「寧寧今日小語」、狀態頁「今日時間軸」）｜放出暖色（橘金用太省、整體偏綠）｜桌面兜底（body #DDD 灰底廉價）。

**核心**：長輩看不清的是「複雜操作」不是「美」；驚艷全藏在不增操作負擔的層次（形象/微互動/情感時刻/會呼吸的背景）。

## 2. 用戶服務體驗（蕪菁頭 · UX）

**子女端 8、長輩端 5、付費 0。** 情感設計天份高（傳話「親口念給媽」、安心存摺、按讚轉達＝真懂用戶），但「子女拿到會盯著媽的管家（爽）、長輩拿到資訊多/入口藏/字小的 App（卡）」。

🔴 P0 必改：
1. **付費旅程 0 觸點**：子女（決策者）從裝機到看儀表板，完全不知要錢/試用剩幾天 → 商業化為零。
2. **長輩端瘦身成「一個主入口」**：首頁＝寧寧大臉＋「跟我說話」巨鈕；狀態/家人/設定 收進子女端或深層（5 分頁長輩迷路）。
3. **聊聊=主入口、視訊降次要**：現在主次顛倒（視訊大鈕、聊天藏底部 11px FAB）；視訊頁臉太小+4 鈕太複雜→臉放大、只留「掛斷」。
4. **危機/求救入口**：長輩慌張找不到大鈕；子女端要真警報非文字承諾。
5. **空狀態**：Day 1 不顯示假數據（23 天/血壓 128/82），改寧寧第一句問候。
6. **長輩引導字放大**：tab-btn 11px、chat-hint 14px muted、task-sub 對比 3:1（低於 AA 4.5:1）——**長輩最需看清的引導字現在最小**。≥18px+高對比。
7. **onboarding 補「調手機」步**：通知權限/音量/螢幕不休眠（必做非建議）——沒調好＝寧寧呼喚傳不到＝功能等於不存在。

🔴 **邊緣時刻幾乎沒設計**（賣「安心」的命脈）：Day1 空狀態／出錯沒網路兜底／**長輩整天沒回應卻報假安好（比沒有更危險）**／危機當下。「裝潢漂亮的樣品屋、沒演練火災逃生」。

做對要守住：傳話、安心存摺、按讚轉達、子女端「媽媽今天安好」、聊聊頁字幕 23px。

## 3. 代碼健康 + Capacitor 上架技術（卡西法 · CTO 第 2 輪）

**代碼 B+（漂亮乾淨、無要重寫）、距上架 0%（純靜態網頁、連 Capacitor 殼都沒生）。** 真正工作量在「殼工程」＝加法不是重做。

🟢 底子好：showView 單一狀態機+事件委派、CSS token 設計系統、[ENGINE] 接點標註、防禦式 DOM。
🔴 3 真 bug：①`styles.css:434` `--coral-d` 未定義（錄音紅色漸層失效）→ 補 `--coral-d:#C06E2E;` ②`app.js:130` `#askCall` 死綁定（漏了「長輩找家人」入口、呼應 UX 缺口）③`say()` 依賴 WKWebView SpeechSynthesis 不可靠（別讓上架版依賴、全換引擎 TTS）。
🟡 重複碼：125 個 inline SVG、status-bar 複製 8 份、family inline style 27 處 → 進 Capacitor 前抽 sprite + status-bar component（**別上 React，vanilla+一個 build step**）。

🔴 Capacitor → iOS（核心缺口、按依賴序）：
1. 生 Capacitor 專案（package.json + capacitor.config + `npx cap add ios`、appId `tw.peining.app`、webDir 接現有 web/）。
2. **6 原生 plugin**：麥克風+WebRTC🔴／APNs 推播🔴（破靜音需 critical alert entitlement、要特別申請）／HealthKit🔴（無官方 plugin、跌倒偵測 iOS 不直接給）／本地通知🟡／IAP RevenueCat🟢／音訊播放🟡（autoplay 需 user gesture）。
3. WKWebView 適配：100vh→100dvh、加 viewport-fit=cover+safe-area（現缺、tab-bar 會被 Home bar 蓋）、假 9:41 狀態列換真安全區。
🔴 **最高風險先 PoC**：Capacitor WKWebView getUserMedia + gpt-realtime WebRTC 麥克風橋接 = **go/no-go 閘門、排第一**。
🟡 外部 randomuser 頭像 23 次 → 上架必換本地素材（弱網白框+隱私）。

基建最低要件：沐寧自己一套 Modal（fork 引擎+多租戶）｜後端 entitlement+推播排程｜多租戶資料層｜錯誤監控｜CI。**無死路、全是補與適配。**

## 4. Apple 上架審核/政策/隱私（沙利曼 · Gate 5 第 2 輪）

**三重高敏疊加（銀髮弱勢 × 健康 × AI/語音境外）→ 高人工審、裸送必退。8 hard blocker、每個有解。**

🔴 hard blocker（不補不能送/不能上傳）：
1. **Privacy Manifest `PrivacyInfo.xcprivacy`**（2024-05 強制、缺=App Store Connect **拒絕上傳**、連審核都進不去）。
2. App 隱私標籤（Nutrition Label、與實際行為不符=下架級）。
3. 權限 usage description（麥克風/HealthKit×2/相機、寫「為什麼」白話）。
4. **語音送 OpenAI 境外揭露+明確同意**（5.1.2、審核員一翻就退、需 in-app 同意畫面）。
5. 醫療宣稱掃描（名稱/截圖/描述/內文零「治療/診斷/醫療級」、用藥=鬧鐘不講藥效）。
6. 帳號刪除功能（有登入必備）。
7. 隱私政策+EULA URL（訂閱必備）。
8. 擬真真人臉=肖像/深偽雷（**v1 走 2D 整顆拆掉=對**；擬真 v1.5 前備授權鏈）。
🟡 健康資料不得用於廣告/不得存 iCloud｜提醒就醫 disclaimer｜AI 內容防護說明｜弱勢付款由成年子女操作呈現｜失智同意走子女代設｜critical alert 慎用｜Individual 帳號未來轉 Organization｜demo 帳號給審核員。

沙利曼可立刻起草（不待 Edward）：Privacy Manifest 清單｜權限文案｜語音境外同意稿｜隱私政策+EULA 底稿｜醫療宣稱掃描+跑一次｜隱私標籤對照表｜送審備註。建議先做：**醫療宣稱掃描（防退件）+ Privacy Manifest 清單（防連上傳都不行）**。
