# 🛠️ 沐寧 開發任務清單（接續開發用）

> 把設計變成可照著打勾的程式工作。2026-06-26 蘇菲建立。
> 環境：Mac（Node + Capacitor + Xcode）。底層複用 `castle-voice-engine`（七成）。

---

## 階段 0 · 先驗兩個死穴（開工前、最便宜換最大確定）
- [ ] **iOS 推播 → 開 App 行為測**：推播一定到（標準功能），要驗的是長輩會不會點進來。可先用簡單原型試水溫。
- [ ] **擬真 Ditto PoC**（授權已過 = 乾淨可商用）：租雲端 4090（RunPod）測 fps≥25 / 首幀≤500ms / 並發上限。約 NT$1-2.5K、2 天。**先查 LICENSE 原檔再跑。**

## 階段 1 · 站在引擎上、做「會跑的最小版」
**殼 + 引擎**
- [ ] 建 Capacitor 專案、把網頁核心包成 iOS 殼
- [ ] 接 `castle-voice-engine`（即時語音 + 會說話的臉 + 記憶 + 登入現成）
- [ ] 上真機跑通、**釘死語音延遲**（命脈）

**反射腦（對話）**
- [ ] 即時聽與講：中文（台灣）優先，英文第二優先；台語暫不列入 v1/v2 承諾
- [ ] 自然節奏：被打斷「聽到→判斷→可能補半句→接話記得上文」

**一張臉（先 2D 保底）**
- [ ] 2D 角色（Audio2Face + Live2D、自架、RTX 3070+ 可跑）
- [ ] 嘴型同步

**推播層（App 沒開時找長輩）**
- [ ] iOS 原生推播：日常問候 / 用藥 / 提醒
- [ ] 推播帶快速鈕（「吃了」/「晚點」不開 App 就回）、緊急可強制響

**五塊 MVP 功能**
- [ ] 用藥提醒（純鬧鐘：到點語音念 + 紀錄吃沒吃 + 漏服通知子女）— **零藥/劑量建議**
- [ ] 家人儀表板（子女看：聊了沒 / 藥吃了沒 / 情緒摘要）
- [ ] 子女代裝流程（≤5 步、15 分鐘 + 手機通知/音量檢查清單）
- [ ] 管家腦 v1：記得上次說的話 + 清晨備好「今天」（天氣/重要日子先做）

## 階段 2 · 養深 + 擬真上場（Ditto PoC 過才做擬真）
- [ ] 管家腦感知層長全（新聞/回診/家人動態、串資料來源）
- [ ] 管家腦記憶層長全（喜好/人格/家人/作息/生命故事）
- [ ] 擬真 4 個臉（2男2女、自架規格依 PoC 結果）
- [ ] 守護腦安全網（情緒/作息/危機關鍵詞 → 通知家人 / 轉介 1925 / 119）

## 階段 3 · 上架前
- [ ] App Store 文案全程守紅線（鬧鐘≠藥師、陪伴≠諮商治療、危機=轉介）
- [ ] Apple 審核送件（Apple 開發者帳號需先辦好）

## 自架基礎（貫穿各階段、不串第三方付費）
- [ ] 聽：中文（台灣）/英文 STT 或即時語音模型接點；不自研語言模型
- [ ] 說：中文（台灣）/英文 TTS 或即時語音模型接點；不把台語列為近期工程承諾
- [ ] 臉：avatar render 自架（2D 現可、擬真待 Ditto）
- [ ] 記憶儲存 + 感知資料來源（天氣/新聞/行事曆）

## 紅線（每個任務都要守）
- 用藥 = 鬧鐘、不是藥師（碰個別用藥建議 = 變醫療器材）
- 情緒陪伴不稱「諮商/治療」
- 危機 → AI 是轉介者、不自己扛
- 全自架、不串第三方每分鐘付費服務
