## 「規劃」的五步（Edward 原話 → 落實）

### 1. 記得根（不漂）
團隊要**記得產品的遠景、目標方向、以及老闆（Edward）講過、要重視、定義好的產品框架與需求**。依這些「根」長出樹幹與樹枝。
- 根 ＝ `產品遠景-核心目標.md`（北極星，Edward 親訂）＋ `design-brief-peining.md`（設計簡報）＋ Edward 歷次拍板。
- 鐵規矩：**任何設計／開發／改版動手前，先讀根。**

### 2. 像產品＋市場經理一樣看外面
持續看／調查**市場、競品、技術、產品、設計**的趨勢、應用與情報。

### 3. 分析 → 長出規格
透過分析，寫好整體產品的：**規格（SPEC）＋ 架構設計 ＋ 服務設計 ＋ 商業模式 ＋ 排好的 backlog**。讓後續**設計、開發、QA** 都知道怎麼安排。

### 4. Codex 交叉審
規格寫完，**同步請 Codex 仔細閱讀** → 檢查、交叉建議、正確度、前瞻度，提供她的洞見與規劃 → **交叉思考整合**進文件。

### 5. 整理 ＋ 路由（workflow）
把**報告／資料／資料夾整理好**，並把**「路由／workflow」設計好**——確保 agent 作業時都能**正確調用文件**；即便修改、迭代，都知道用**正確的流程與邏輯**把產品規劃好。
**核心戒律：不是單純「講一個修一個」。**
