## 二、資料模型定案（最小增量：3 張新表＋persons 加 3 欄）
- **家庭圈＝現有 account**（不另造租戶）：加人＝同帳號下多一組 person＋成員＋家庭成員。
- `persons` 加：`auth_user_id`（登入者↔家中角色的連結、可空——長輩未領帳號/小孩純檔案）、`region_code`（餵天氣）、`attributes`（遊戲份額/頭像/是否未成年）。
- 新表 `family_invitations`：token（LINE 深連結/QR）＋ **6 位數字碼**（長輩電話裡念得出）＋ `elder_assisted` 模式（子女預建阿嬤檔案、長輩認領）＋ 72h 效期/撤銷。
- 新表 `consent_records`：同意紀錄（個資法審計＋感知層能力清冊一表兩用；type 如 family_link / share_mood_weather / capability:voice_emotion）。
- 分享開關住現有 `family_memberships.permissions`（jsonb）：`share_mood_weather / share_health / share_memories_family_book`，每人自控。
- 角色沿用現有兩層 enum、不新增；**允許多位被關心者**（全家都是被關心的人）。

## 四、流程定案
1. **邀請**：家人分頁「邀請」→ 產 LINE 分享連結＋6 位碼 → LINE 只當分享通道（登入維持 Apple/Google/OTP）。
2. **一般加入**：點連結/輸碼 → 登入 → 後端驗＋查名額 → 建檔連結 → **自選分享層**同意畫面 → 留同意紀錄。
3. **長輩協助式加入**（差異化重點）：子女預建「阿嬤」檔案＋先設好提醒 → 長輩裝置輸 6 位碼認領 → **寧寧語音同意**（白話唸、大按鈕「好，我同意」）→ 生效。
4. **離開/移除/轉移**：自行離開、owner 移除、owner 轉移；長輩資料走既有隱私匯出/刪除。
5. **多重家庭**：V1 一人一圈（主流三代同圈已涵蓋）；P2 用「派生快照跨圈推送」開三明治世代雙圈、不開跨帳號查詢。

## 五、引擎資料流變更
- **最大工程點**：後端從「環境變數釘死一人」→ **每個請求由登入身分解析出「這是誰」**（token→person→account 的範圍化配接）。
- 心情訊號 wellbeing_signals 上雲（新表）；照護行事曆併入既有 `routine_reminders`（省一張表）。
- **家人狀態感知源（感知層最後 5%）＝派生快照**：背景整理員對每位成員產一行「已過濾的可見摘要」（天氣級＋一句觀察）→ 存 family_context 快照（表已在）→ ① 家人分頁讀這個 ② 寧寧脈絡注入 → 才能說「美華最近睡比較少喔，要不要我幫你留句話？」。家人端**永遠只讀快照、不讀原始訊號**。
- 互動鈕（愛心/語音留言）→ 既有 family_interaction_events → 寧寧下次開場轉達。
